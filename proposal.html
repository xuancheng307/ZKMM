1問題定義

1.1 什麼是零知識證明？為什麼跟數學函數有關？

零知識證明 (Zero-Knowledge Proof, ZKP) 讓一方（證明者）向另一方（驗證者）證明 「我知道某個計算的結果」，但不洩漏計算過程中的任何秘密資訊。

實際應用場景：

  • 隱私機器學習推論：模型擁有者證明「我用這個模型對你的資料做了推論，結果是 X」，但不洩漏模型參數
  • DeFi / 金融：證明「我的資產滿足某個條件」，但不公開具體數字
  • 可驗證計算：雲端伺服器證明「計算結果確實正確」，客戶端不需要重新計算

這些應用中，模型推論需要計算啟動函數 (activation functions)，例如：

函數用途數學定義
exp(x)Softmax, 注意力機制ex
ln(1+x)對數損失, 特徵轉換自然對數
erf(x)GELU 啟動函數的核心(2/√π) ∫ e-t²dt
GELU(x)Transformer 標準啟動函數x · Φ(x)，其中 Φ 涉及 erf

1.2 問題的核心：ZK 電路沒有浮點數

ZK 電路在有限域 (finite field) 上運算。以 BN254 曲線為例，所有計算都在一個 約 254 位元的質數域 Fp 中進行，其中 p ≈ 2254。

核心困難：有限域只支援整數的加法和乘法。沒有浮點數、沒有除法（只有乘法逆元）、沒有 math.exp()。 要在這樣的環境中計算 exp, ln, erf，需要把這些函數「翻譯」成只用整數加法和乘法的形式。

1.3 現有做法及其問題

做法一：查表法 (Lookup Table)

最直觀的方法：預先計算所有可能的 (輸入, 輸出) 對，存成一張表。 電路中只需要一次查表 (lookup) 就能得到結果。

優點
  • 精度完美 (ULP = 0)
  • 電路結構簡單（1 次查表）
問題
  • 表格巨大：16-bit 精度需要 65,536 筆
  • 每個函數需要各自的表
  • 證明時間 ~1,200 ms（表格載入成本高）

做法二：Taylor 多項式

用 Taylor 級數在某一點展開，取有限項作為近似。 例如 ex - 1 ≈ x + x²/2 + x³/6 + x4/24 + x5/120。

優點
  • 只需 256-entry 的輔助表（byte range check）
  • 電路結構通用
問題
  • 精度差：exp 的 max ULP = 12
  • ln 更慘：max ULP = 762
  • erf 最慘：max ULP = 1,467
  • Taylor 在展開點附近精確，但遠離展開點時誤差快速增長

1.4 我們的目標

研究問題： 能不能設計一個方法，用跟 Taylor 一樣小的表 (256 entries)，但精度接近查表法？ 而且這個方法要能推廣至 13 個函數（exp、ln、sin、erf、GELU、sigmoid 等）？

2背景技術

2.1 零知識證明中的算術電路

ZK 證明系統（如 PLONK、Halo2）將計算表達為算術電路：由加法門 (addition gates) 和乘法門 (multiplication gates) 組成的有向無環圖。電路的每條「線」(wire) 攜帶一個 有限域 Fp 中的元素。

電路有兩種輸入：

  • Public input：驗證者知道的值（例如：函數輸出）
  • Private input (witness)：只有證明者知道的值（例如：中間計算步驟）

Lookup argument（查表論證）是現代 ZK 系統的重要功能： 允許電路宣稱「某個值存在於某張預定義的表格中」。這讓我們可以用查表來替代複雜的 算術約束。例如：驗證一個值在 [0, 255] 範圍內，用 lookup 比用 8 個 bit 約束更高效。

2.2 有限域中的固定點算術

為什麼需要固定點？

有限域只有整數。要表達小數（例如 ln(1.5) = 0.405...），我們使用固定點表示法： 選定一個縮放因子 S = 2pp，其中 pp 是精度參數 (precision parameter)。

以 pp = 16, S = 65536 為例：

真實值固定點表示計算方式
0.532,7680.5 × 65,536
0.693 (= ln2)45,4260.6931... × 65,536 取整
0.843 (= erf(1))55,2270.8427... × 65,536 取整
1.065,536= S（超出 16-bit 範圍）

精度指標：ULP (Unit in the Last Place)

ULP 是固定點算術中的標準精度指標：

ULP 定義： ULP(R) = |wcircuit − wreference|
其中 wreference = round(f(R/S) × S) 是理想的固定點結果。

ULP = 0 表示完美精度；ULP = 1 表示差了最小刻度的 1 格； pp = 16 時，ULP = 1 相當於約 15 bits 的有效精度。

截斷問題 (Truncation)

固定點乘法會產生「多餘的精度」。例如兩個 pp=16 的數相乘：

(a × S) × (b × S) = ab × S²

結果的 scale 變成 S²，需要除以 S 才能回到正確的 scale。在有限域中，這個「除以 S」的操作稱為截斷 (truncation)：

result = (a × b × S²) / S = ab × S + remainder

截斷會丟失 remainder（餘數），這是固定點算術的精度損失來源之一。

2.3 多項式近似

Taylor 級數：在一點展開

Taylor 級數在 x = 0 處展開函數，取前 d 項作為近似：

ex − 1 ≈ x + x²/2! + x³/3! + x4/4! + x5/5!

Taylor 的特性：在展開點 (x=0) 附近非常精確，但隨著 x 遠離展開點，誤差快速增長。 更精確地說，Taylor 最小化的是局部誤差（高階導數），不是整個區間上的最大誤差。

Minimax 近似：最小化最大誤差

Minimax（也稱 L∞ 最優）近似找到的是在整個區間 [a, b] 上最大誤差最小的多項式：

P* = argminP maxx∈[a,b] |f(x) − P(x)|

Taylor（局部最優）

在 x=0 完美，在區間邊緣誤差大

誤差分佈：[小 小 小 中 大 大 大]

Minimax（全域最優）

誤差在整個區間上均勻分佈（等振盪 equioscillation）

誤差分佈：[中 中 中 中 中 中 中]

為什麼 Minimax 天然適合 ZK： ZK 電路的精度指標是 max ULP（最壞情況），這正好是 L∞ norm。 Taylor 最小化的是局部誤差，但 ZK 需要的是全域最壞情況保證。 Minimax 直接最小化我們關心的指標。

Remez 演算法是計算 Minimax 多項式的標準方法。輸入函數和區間， 輸出最優的多項式係數。這是一個成熟的數值方法（1934 年提出）， 在 FHE（全同態加密）領域已廣泛使用。ZKFixedPointChip 也使用了 Minimax（via lolremez），但缺乏正式誤差分析和 Taylor 基線對照。

2.4 Horner 求值法

Horner's method 是 sequential 多項式求值中乘法次數最少的方式（d 次，理論下界）。對於 P(x) = c5x5 + c4x4 + ... + c1x：

P(x) = ((((c5 · x + c4) · x + c3) · x + c2) · x + c1) · x

每一步只需要一次乘法 + 一次加法。d 次多項式只需要 d 步 Horner iteration。 在 ZK 電路中，每一步對應一個 Horner gate（算術約束）：

accnext = acccurrent × R + Di

其中 R 是輸入（固定點表示）， Di 是預先計算好的係數常數。

2.5 現有的查表方法

Method A：直接查表 (Direct Lookup)

預計算完整的 (R, w) 表格。R 是輸入，w = round(f(R/S) × S) 是對應的輸出。 表格大小 = S = 65,536 筆。電路中一次 lookup 搞定。

精度：ULP = 0（完美），但表格巨大，prove time ~1,200 ms。

Method B：分塊查表 (Chunk Decomposition)

利用 exp 的代數性質 ea+b = ea × eb， 將輸入拆成高低位元組查兩張小表，再乘起來。表格只需 768 筆。

Method B 的限制： 只適用於 exp。ln 的對應性質 ln(a×b) = ln(a) + ln(b) 無法用在加法輸入上。 erf 完全沒有類似的分解性質。
對於 ln 和 erf，多項式近似是唯一的小表方案。

3.1 Step 1: 區間選擇 + Output Range Verification (ORV)

區間選擇的 trade-off

多項式近似在越窄的區間上精度越好。但區間越窄，覆蓋的輸入範圍越小， 可能需要額外的 range reduction 電路。這是一個設計決策：

我們選擇窄區間，優先保證精度。

ORV：為什麼不需要 explicit range check

傳統做法：電路中需要一個約束 R ≤ R_MAX，確保輸入在合法範圍內。 這需要額外的 gate。

我們的觀察：如果近似多項式 P 在 [0, R_MAX] 上是單調遞增的 （這對逼近單調函數的低誤差多項式自然成立）， 而且輸出 w = ⌊P(R)/Sd⌋ < S，那麼：

ORV 核心定理：
對於單調遞增的多項式，如果輸出 w < S， 那麼輸入 R 一定在 [0, R_MAX] 內。

Regime 1 — 近距離越界（R* ∈ (R_MAX, p1/d]）：
此時 P(R*) 作為整數運算（無模運算 wrap-around，因 R* < p1/d ≈ 250.8）， 由單調性 P(R*) > P(R_MAX) ≥ Sd+1， 故 w = ⌊P(R*)/Sd⌋ ≥ S，ORV 確定性拒絕。 此區間內偽造成功的機率為 0——無機率論證，純粹基於單調性。

Regime 2 — 遠距離越界（R* > p1/d）：
P(R*) mod p 發生 wrap-around，結果在 Fp 中偽隨機分佈。 degree-d 多項式對任意目標值最多有 d 個原像， 合法輸出值域 [0, Sd+1) 中最多有 d · Sd+1 個 R* 能產生合法 w。 偽造成功的機率 ≤ d·Sd+1/p = 5·296/2254 ≈ 2−156（d=5, pp=16）。

結果：只需要檢查「輸出 w < S」（一次 byte decomposition）， 就同時驗證了輸入範圍和輸出有效性。省掉一個 explicit range check gate。

3.2 Step 2: Minimax 係數取代 Taylor

這是對 §2.3 Taylor 的直接改進。電路結構完全相同（都是 Horner 求值）， 唯一的差異是把 Taylor 係數換成 Minimax（Remez 最優）係數。

為什麼改善這麼大？

以 ln(1+x) 為例，Taylor 的 5 次展開：

ln(1+x) ≈ x − x²/2 + x³/3 − x4/4 + x5/5

這些係數 (1, -1/2, 1/3, -1/4, 1/5) 是由 ln 在 x=0 的導數決定的。 它們是數學上固定的，完全沒有考慮區間 [0, ln2) 的特性。

Minimax 的係數是針對區間 [0, ln2) 最佳化出來的，允許在 x=0 附近稍微 犧牲精度，換取區間邊緣的大幅改善。結果：

d* 差距的根本原因

為什麼 Taylor 在多數配置中無法達到 ULP≤1？兩個結構性原因解釋了這個系統性差距：

原因一：對稱零項浪費係數。 奇函數的 Taylor 展開只含奇數次項——偶數次係數恆為零。 一個 degree-5 的 Taylor 多項式實際只使用 3 個非零項（deff = 3）， 而 Minimax 的 5 個係數全部是自由參數。 收斂快的奇函數（sin, sinh）靠 1/n! 衰減仍可在 d≤9 達標，但 收斂慢的奇函數（erf, arcsin, tan）則被推至 d*>9。 例如 arctan 在 pp=16 時，Minimax d*=4 而 Taylor d*=9——差距 5 次主要來自零項浪費。

原因二：收斂緩慢或收斂半徑不足。 ln(1+x) 的 Taylor 係數僅以 1/n 衰減（非 1/n!）， sigmoid 的收斂半徑僅 π ≈ 3.14， erf 等複合函數的高階係數衰減受限於內層函數結構。 不具對稱性的 ln、sigmoid、GELU、1-exp(-x)、1-exp(-x²) 同樣因收斂不足而 d*>9。 8/13 無法達標的函數各受至少一個原因（部分兩者兼具）的強烈影響。

關鍵洞見： 兩個結構性原因解釋了 8/13 函數的 Taylor d* > 9： (1) 對稱零項浪費一半係數，(2) 收斂緩慢或收斂半徑不足。 每個無法達標的函數至少受其一影響（erf、arcsin、tan 兩者兼具）。 Minimax 避開了這兩個限制——所有係數均為自由參數，不受 Taylor 結構約束。

多項式次數直接決定 Horner 電路的大小。以 sigmoid pp=16 為例：

3.3 Step 3: Deferred Truncation

這是對 §2.2 截斷問題的改進。

Per-step Truncation（傳統做法）

Horner 每一步都做截斷，把中間值限制在固定精度：

step 1: acc = trunc(c5 × R + c4)  ← 截斷 1
step 2: acc = trunc(acc × R + c3)   ← 截斷 2
step 3: acc = trunc(acc × R + c2)   ← 截斷 3
step 4: acc = trunc(acc × R + c1)   ← 截斷 4
step 5: w = trunc(acc × R)          ← 截斷 5

每次截斷都丟失精度，5 次截斷的誤差累積。

Deferred Truncation（我們的做法）

利用 BN254 有限域有 254 bits 的容量，而 d 次 Horner 的中間值最多只需要 d×pp bits（例如 d=5, pp=16 時 ~96 bits，遠小於 254）， 因此可以在整個 Horner chain 中完全不截斷，讓精度一直保持到最後才做一次截斷：

step 1: acc = D5 × R + D4       ← 不截斷
step 2: acc = acc × R + D3       ← 不截斷
step 3: acc = acc × R + D2       ← 不截斷
step 4: acc = acc × R + D1       ← 不截斷
step 5: P(R) = acc × R            ← 不截斷
final : w = P(R) / Sd           ← 唯一一次截斷

關鍵洞見： BN254 的域足夠大（254 bits），能夠容納整個 Horner chain 的中間值（d×pp bits）而不 overflow。 這讓我們可以把 d 次截斷合併為 1 次，大幅減少精度損失。 可行的 (pp, d) 組合必須滿足 d×pp < 253（BN254 約束）。

精度改善（全部 R 值窮舉驗證）：

函數Deferred ULPPer-step ULP改善
exp(x)-1122.0x
ln(1+x)122.0x
erf(x)362.0x

三個函數一致地獲得 2.0x 改善。代價是最後的截斷需要 10 個 byte lookup 來驗證餘數。

5範圍界定與已知限制

5.1 論文範圍

本論文聚焦於綠色方框：Horner evaluation kernel + ORV。 Range reduction（將任意輸入映射到多項式區間內）和 reconstruction（將結果組合回去） 是function-specific 的應用層電路，不屬於本論文的通用方法論。

ORV 使得 kernel 本身是自洽的：合法輸入得到正確輸出，非法輸入被 ORV 拒絕。 當 ORV 適用條件成立時（見 §3.1），不需要在 kernel 內部做 explicit range check。 對於 ORV 不適用的 (function, pp, d) 組合，可使用 Explicit Range Check 變體（+1 gate, +1 row）， 電路架構的其餘部分（Horner kernel + deferred truncation）完全不變。

6結論與預期貢獻

三個預期學術貢獻

  1. Minimax 係數選擇：據我們的文獻回顧，目前還沒有在 ZK Horner 電路中系統性地使用 Remez 最優多項式的工作。 d* 比較顯示：52 組 (函數 × pp) 配置中 33 組 Taylor 無法在 d≤9 內達到 ULP≤1 而 Minimax 可以； 8/13 函數 Taylor 在所有精度等級下均無法達標。5 個兩者達標的函數中 Minimax 平均節省 2.9 次（見 §3.2 表格）
  2. Output Range Verification (ORV)：新技術——利用單調性做 implicit input validation，省去 explicit input range check gate——w 的 byte 分解即隱含驗證輸入範圍（仍使用 256-entry ByteTable）。適用條件：多項式輸出 w ∈ [0, S)（由區間設計與 degree 選擇保證，窮舉驗證確認）。13 個函數中 11 個在所有配置適用（旗艦 pp=16 d=5 下 12/13，d≥6 下 13/13），其餘透過提高 degree 解決
  3. Deferred Truncation：利用 BN254 大域容量，將多次截斷合併為一次，精度改善 2x

(pp, d) 設計空間

本方法的核心優勢之一是 pp（精度）和 d（多項式次數）均為可配置的設計參數。 使用者可根據精度需求和電路預算，從設計空間中選擇最適合的組合。 以下為 13 個函數在不同 (pp, d) 配置下達成 ULP≤1 的覆蓋情況（pp≤28 為窮舉驗證）：

一套在 ZK 電路中求值超越函數（exp、ln、erf、sin、...）的完整方法論。 在金融級應用場景中實現最高 44 個有效位元的精度， 證明大小僅 7–15 KB，驗證時間為毫秒級。

1 研究貢獻定位

精確界定本研究做了什麼、沒做什麼

不是我們的貢獻

「我們用 Remez 算法找到了更好的多項式係數。」

Remez 只是工具 — 就像 halo2 是工具一樣。Remez 交換算法自 1934 年起已是數值分析的標準方法。

是我們的貢獻

一套完整的方法論：在零知識電路中實現高精度超越函數求值 — 涵蓋電路設計、數值分析、以及端到端的金融模型證明。

1
通用 Horner 核心電路 一個參數化的 halo2 電路元件（3 組自定義閘、通用 ByteTable[0..255]）。切換函數只需替換 d 個係數 — 不需重新設計電路。支援 13 種超越函數。
2
截斷策略分析 首次發現在 pp≥32 時，延遲截斷的中間值會溢出 BN254 有限域（253 位元），使逐步截斷成為高精度 ZK 算術的唯一可行路徑。
3
設計空間地圖 637 組 (pp, d) 配置橫跨 13 種函數的系統性實驗 — 包含「量化飽和邊界」：多項式次數超過某值後，精度反而下降而非提升。
4
係數條件數效應（新發現） 首次揭示函數的係數結構造成相同參數下 10 倍精度落差。在 ZK 定點數文獻中尚無先例。
5
端到端金融模型 ZK 證明 從抽象方法論到可運行的 KZG 證明：Black-Scholes 選擇權定價（場景 A）與 Merton 結構模型違約機率（場景 C）。

2 場景 A — Black-Scholes 選擇權 Delta 合規證明

證明投資組合風險在合規範圍內，不透露任何持倉細節

場景 A — 基線驗證

背景故事

一家對沖基金在交易所持有大量選擇權部位（call 和 put 混合）。監管機構要求基金定期報告風險暴露，確認投資組合的 Delta 絕對值不超過門檻（例如 0.05，接近 delta-neutral）。

傳統做法是把完整持倉清單交給監管機構 — 但這等於公開交易策略。對沖基金的策略是它的命脈：一旦競爭對手知道部位結構，就能搶先交易（front-running），讓建倉成本大幅上升。

基金的目標：在不透露任何資訊（包括操作哪支股票、持有多少部位）的情況下，向監管機構證明自己是合規的。

角色分工

角色身份職責
證明者對沖基金持有選擇權部位，想保護交易策略。生成 ZK 證明
驗證者監管機構確認 |Delta_total| ≤ 0.05，毫秒級驗證，不需知道持倉
可信第三方清算所記錄所有交易，每日公布持倉的 Merkle root（一個 hash），保證輸入真實性

為什麼需要超越函數

計算選擇權 Delta 需要 Black-Scholes 公式。這個公式自然包含三種超越函數：

  • ln(S/K) — 計算 moneyness，衡量選擇權是價內還是價外
  • exp(-qT) — 計算股息折現因子
  • erf(d1/√2) — 計算常態分布 CDF Φ(d1)，Black-Scholes 的核心

這三種函數都不是加減乘除能算的。ZK 電路只能做有限域上的加法和乘法，所以必須用特殊方法來處理。

電路架構

┌────────────────────────────────────────────────────┐ │ 公開輸入: Merkle_root, threshold │ │ 私密見證: S, r, sigma_i, n, 所有部位資料 │ └───────────────────┬────────────────────────────────┘ │ ┌───────────────v──────────────────────┐ │ 對每張選擇權 i（共 N 張） │ │ │ │ Part A: Merkle 驗證 ← 已有技術 │ │ Part B: Delta 計算 │ │ B2. ln(S/K) ★ 本論文方法 │ │ B12. erf(d1/√2) ★ 本論文方法 │ │ B15. exp(-qT) ★ 本論文方法 │ │ Part C: 累加 contrib ← 標準加法 │ └───────────────┬──────────────────────┘ │ ┌───────────────v──────────────────────┐ │ |Delta_total| ≤ threshold → 合規 │ └──────────────────────────────────────┘

三種超越函數共用同一個 ByteTable（256 行），只換係數向量。每張選擇權呼叫 3 次超越函數，n 張共 3n 次。

計算步驟（每張選擇權 17 步）

  1. ratio = S / K_i — 除法（witness + 驗證）
  2. log_ratio = ln(ratio) — 超越函數 #1
  3. vol_sq = σ_i × σ_i — 乘法
  4. drift = r - q_i + vol_sq/2 — 加減法
  5. numerator = log_ratio + drift × T_i — 乘法 + 加法
  6. d1 = numerator / (σ_i × √T_i) — 除法
  7. erf_val = erf(d1 / √2) — 超越函數 #2
  8. Φ(d1) = (1 + erf_val) / 2 — 加法 + 乘法
  9. discount = exp(-q_i × T_i) — 超越函數 #3
  10. Δ_i = discount × Φ(d1) — 乘法（call 的情況）

3 場景 C — Merton 結構模型違約機率合規證明

證明投資組合違約機率在監管門檻以下，不透露持倉組成

場景 C — 高精度擴展

背景故事

一家信用衍生品交易商持有 N 筆信用違約交換（CDS）部位，每筆對應不同公司。監管機構要求交易商定期報告投資組合的加權違約機率（weighted PD），確認在可接受門檻以下（例如 5%）。

傳統做法是交出完整持倉清單 — 但這等於公開哪些公司的信用風險被交易、曝險多大。投資組合組成是交易商的核心商業機密。

交易商的目標：在不透露任何持倉資訊的情況下，向監管機構證明組合風險在合規範圍內。

角色分工

角色身份職責
證明者信用衍生品交易商持有 CDS 部位，保護組合結構。生成 ZK 證明
驗證者監管機構確認 PD_portfolio ≤ threshold，毫秒級驗證

為什麼需要高精度

精度需求

投資等級公司的違約機率在 0.001–0.01（10–100 個基點）。在這個量級下，即使小的絕對誤差（例如 ±0.002）就可能翻轉合規/不合規的判定。場景 A 的 pp=24 對 Delta 足夠，但對違約機率只是勉強 — 場景 C 測試 pp=32 和 pp=48 能否為更嚴格的 Merton 模型提供足夠精度。

Merton 結構模型

公司在到期日違約，當且僅當資產價值低於負債：VT < D。假設資產價值服從幾何布朗運動，違約機率為：

PDi = Φ(−d2i) = (1 − erf(d2 / √2)) / 2

其中：

d2 = [ ln(V/D) + (r − σ²/2) × T ] / (σ × √T)

所需超越函數

  • ln(V/D) — 計算資產對負債比率的對數，衡量公司的緩衝空間
  • erf(d2/√2) — 計算常態分布 CDF，將 d2 統計量轉換為違約機率

與場景 A 相比：場景 C 只需 2 種函數（無 exp），但精度要求從 pp=24 提升到 pp=32/48。

電路架構

┌──────────────────────────────────────────┐ │ 公開輸入: r, T, threshold │ │ 私密見證: V_i, D_i, σ_i, w_i, N │ └──────────────┬───────────────────────────┘ │ ┌──────────v───────────────────────┐ │ 對每筆部位 i（共 N 筆） │ │ │ │ Part A: ln(V/D) 管線 │ │ LeadBit 範圍歸約 + ★ Horner │ │ Part B: Merton 算術 │ │ σ², drift, d2 計算 ← 標準閘 │ │ Part C: erf(d2/√2) 管線 │ │ SignExtend + ★ Horner │ │ Part D: PD 計算 + 加權累加 │ └──────────┬───────────────────────┘ │ ┌──────────v───────────────────────┐ │ PD_portfolio ≤ threshold → 合規 │ └──────────────────────────────────┘

計算步驟（每筆部位 13 步）

  1. ratio = V_i / D_i — 除法
  2. ln_val = ln(ratio) — 超越函數 #1
  3. σ_sq = σ_i × σ_i — 乘法
  4. drift = (r − σ_sq/2) × T — 減法 + 乘法
  5. numerator = ln_val + drift — 加法
  6. d2 = numerator / (σ_i × √T) — 除法
  7. erf_val = erf(d2 / √2) — 超越函數 #2
  8. PD_i = (1 − erf_val) / 2 — 減法 + 乘法
  9. contrib_i = w_i × PD_i — 加權

為什麼必須用逐步截斷

關鍵約束

延遲截斷在 Horner 鏈中不做中間截斷，讓中間值累積。但 pp=32 時中間值達 ~2347 位元，pp=48 時達 ~2713 位元 — 遠超 BN254 有限域的 253 位元容量。逐步截斷是 pp≥32 的唯一可行策略。

係數條件數效應 — 本實驗最重要的精度發現

ln 和 erf 使用完全相同的核心、相同精度、相同次數 — 但精度差了 9.4 倍。原因是 ln(1+x) 的 14 個 Minimax 係數全部有顯著幅度且交替正負，而 erf(x) 作為奇函數，約一半的偶數次係數接近零，等效只有 ~7 個活躍 Horner 步驟。在 ZK 定點數文獻中，這個發現尚無先例。

多項式方法 vs 查找表（pp=32 配置）

查找表方法

每種函數需要 232 = 43 億行。兩種函數合計 86 億行，所需 k ≥ 34。

完全不可行。Halo2 電路的 k 通常在 15–22 之間。

多項式方法

ByteTable = 256 行（不變）。ln 核心 = 12 行 + 48 lookups。erf 核心 = 12 行 + 48 lookups。

在高精度場景下，多項式方法是唯一的選擇。

分段 erf 擴展定義域

解決方案

單段 erf 逼近區間為 [0, 1]，只能處理 d2/√2 < 1 的投機級公司。為覆蓋完整信用評等光譜（CCC 到 AAA），我們實作了 5 段分段 Minimax 多項式，將 erf 定義域擴展到 [0, 5]。每段使用獨立的 Minimax 係數組和預計算偏移量 erf(0)–erf(4)，共用同一個 ByteTable。電路使用 5 列 × 4 行的 seg_mux 閘實現分段選取。

覆蓋範圍：d2/√2 ≤ 5 → 涵蓋 CCC 到 AAA 全光譜。第 5 段 [4,5] 信號僅 ~66 ULPs（SNR 12.7x），但該區間 PD 在 pp=32 精度下本質為 0，不影響結果正確性。

pp=32 的實用性結論

pp=32（28.6 有效位元）在實際信用風險場景中已足夠使用。搭配 5 段分段 erf，ZK 管線的 PD 輸出在全部 7 個評等等級（CCC 到 AAA）上與解析解完全一致到 8 位小數。pp=48 的核心層級測試（40.8–44.1 有效位元）展示了方法的精度擴展潛力，但對當前應用場景而言 pp=32 已足夠。

erf 定義域上界 — 已解決。已實作 5 段分段 Minimax 多項式，覆蓋 erf 輸入 [0, 5]（對應 d2/√2 ≤ 5），涵蓋 CCC 到 AAA 完整評等光譜。第 5 段 [4,5] 信號僅 ~66 ULPs（使用降階 d=6 多項式，SNR 12.7x），但該區間 PD 本質為 0，不影響正確性。d2/√2 > 5 的極端情形（理論上可能但極罕見）會飽和在 erf(5) ≈ 1。

2

pp=48 完整電路尚未建構。pp=48 d=14 的結果為核心層級基準測試（隔離的 Horner 求值）。完整 Merton 電路的端到端整合尚未完成，約束數量和證明時間可能與核心外推不同。

3

Debug 模式證明時間。場景 C 的 ~30 秒證明時間（5 段分段 erf）是在 Rust debug 模式下測量的。Release 模式（含編譯器優化）預計顯著更快，但尚未正式測量。

4

投資組合批次處理未實作。目前實作對單一 (V, D, σ) 元組產生一個證明。將多筆部位批次打包進單一證明 — 這是真正的組合層級證明所必需的 — 在核心設計上已支援，但尚未實作。

所有實驗以 Rust 實作（halo2 + KZG）。 測試：51/51 全部通過。 係數由 Python mpmath Remez 交換算法以 80 位精度生成。

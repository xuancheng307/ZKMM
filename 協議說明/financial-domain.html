跨樹配對：MDT 的前 4 個欄位（index 0-3）是配對鍵，與 Position leaf 的欄位 (4,1,2,3) 一一對應。Proof #1 在電路內強制這 4 個值相等，確保每個持倉讀取的是正確的市場數據。
待更新：隱含波動率 σ 計劃從 Proof #1 witness 移入 MDT 作為第 9 個欄位。σ 是公開可驗的（交易所和數據商都公布），放在 MDT 中可省去電路內的 Newton-Raphson 反推，安全性不變。此變更尚未實施。

2.3 Trade Action Tree（交易動作樹）

每葉存放一筆交易指令及其 EdDSA 簽名。Proof #2 在電路內驗證簽名真偽後套用到 State Tree。

4交易動作詳解

4.1 TRADE (000) — 一般交易

涵蓋所有交易所授權的狀態變更：買入、賣出、部分平倉、換倉。交易所用 BabyJubjub EdDSA 簽名每筆交易指令。

典型場景：基金在交易所買入 100 股 AAPL，交易所簽名 → w 從 0 變成 +100。

4.2 EXERCISE (001) — 期權行權

持有人行使期權權利。行權後期權合約消滅（expiry → 0），可能產生標的股票的交割（由另一筆 TRADE 處理）。

典型場景：行使 10 張 AAPL Call → expiry 歸零，另外一筆 TRADE 記錄買入 1000 股 AAPL。

4.3 CLOSE (010) — 平倉 / 強制平倉

倉位完全關閉，w → 0。用於主動平倉、融資維持率不足強制平倉等場景。

4.4 EXPIRY (011) — 到期失效

價外期權到期自動失效，w → 0。電路可透過 today ≥ expiry_date 驗證到期事實。

設計待優化：目前 EXPIRY 仍需 EdDSA 簽名。理論上到期是客觀事實（電路可自行驗證），不應需要外部簽名。此為已知待改進項目。

4.5 CORPORATE_ACTION (100) — 公司行動

非交易所交易的狀態變更（拆股、股利等）。不使用 EdDSA，改由 Event Tree Merkle proof 授權。詳見 Section 5。

4.6 NOP (111) — 無操作

填充用。當一個結算期的交易數少於 batch size 時，用 NOP 填滿。所有欄位保持不變。

5公司行動 (Corporate Actions)

公司行動是由上市公司或監管機構發起的事件，會影響持倉狀態但不經過交易所交易流程。協議使用 Event Tree 作為統一機制處理這類事件。

5.1 Group A：比率調整（Phase 1 實作中）

電路約束：new_w = old_w × ratio, new_K × ratio = old_K

電路是 event_type 無關的 — 四種事件使用完全相同的比率約束。

6不影響持倉狀態的事件

以下金融事件不改變 Position State Tree 的任何欄位，因此協議不需要處理：

事件為什麼不影響持倉
一般現金股利 (Regular Cash Dividend)現金流入帳戶，不改變 w, K, expiry 或 underlying
成本基礎調整 (Cost Basis Adjustment)稅務紀錄，不影響持倉狀態
帳戶轉移 (ACAT Transfer)託管人變更，持倉不變
交易暫停 (Trading Halt)暫時性，不改變任何持倉欄位
更名 / 換 Ticker (Name Change)只是標籤變更，underlying_id 是數字 ID 不受影響

8覆蓋範圍總結

8.1 Biconditional Property

State(T) ≠ State(T-1)   ⇔   ∃ valid Trade ∪ Event ∪ Expiry 授權每一處差異

協議追求的核心性質：狀態有變化，若且唯若有合法的交易、事件或到期授權了該變化。

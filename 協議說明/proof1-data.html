Taylor 限制：ln 和 erf 的 Taylor 展開含有負係數（ln: c2=-1/2, c4=-1/4; erf: c3, c7 項為負），在 unsigned ZK 電路（BN254 質數域）中 Horner 中間值會溢位為巨大正數。因此 Taylor 無法用於 ZK 電路的 ln 和 erf 計算。exp 的 Taylor 係數全正（1/k!），理論上可用但精度遠不及 Minimax。

瓶頸步驟：B10 除法將誤差放大 ~20 倍（6.24 → 125.7）。但後續三道「天然煞車」— erf 有界性、Φ 的 0.5 縮放、discount ≤1 的乘法 — 將誤差逐步壓回 ~57 ULP。這不是巧合，而是 BS 公式的結構性質。

這張表最令人困惑的地方：B10 的誤差上界高達 126 ULP，但最終結果卻只有 57 ULP。看起來像是誤差憑空消失了。讓我解釋為什麼。

— 停頓 —

第一個問題：B10 為什麼會放大 20 倍？

B10 計算 d1 = numerator / (σ√T)。分子累積了 ~6.24 ULP 的誤差，而分母 σ√T 可以很小（低波動率、短到期）。除以一個小數 = 放大誤差。具體地：誤差上界 ≈ (分子誤差 / 分母) + (分母誤差 × 值 / 分母²) ≈ 6.24 × 20 ≈ 126 ULP。（指向 B10 紅色列） 這個 20 倍放大是不可避免的 — 它是 BS 公式的數學結構決定的。

— 停頓 —

第二個問題：為什麼不繼續惡化？三道天然煞車。

煞車 1 — erf 的飽和壓縮。 erf(x) 是 S 型函數，值域固定在 [−1, +1]。它的導數 erf'(x) = (2/√π)e−x² — 當 |x| 增大時指數衰減到 0。關鍵觀察：造成 B10 高誤差的輸入（小 σ√T），正好讓 |d1| 很大。而 |d1| 大 → erf 幾乎平坦 → 輸入誤差被壓縮。反過來，erf 導數最大的地方（d1≈0），B10 的實際誤差也小。這個反相關是 BS 公式的結構性質。

但上界分析必須考慮最壞情況。在 d1≈0 附近，erf'≈1.13，所以 B12 上界 = 89.98 × 1.13 + 1.61（kernel 誤差）≈ 103 ULP。（指向 B11→B12） 注意：90 → 103 只是微幅增加，而不是再次 20 倍放大。erf 阻止了誤差的指數增長。

— 停頓 —

煞車 2 — Φ 的 0.5 縮放。 Φ(d1) = 0.5 + 0.5 × erf(d1/√2)。那個 0.5 係數直接把 erf 階段的 103 ULP 砍半：103 × 0.5 + 0.5 ≈ 52 ULP。（指向 B13） 這是純粹的數學定義給的免費壓縮。

煞車 3 — discount ≤ 1。 最終 Δ = e−qT × Φ(d1)。折現因子 e−qT ≤ 1，所以乘法不會放大 Φ 的誤差。合併兩項誤差：52 × 1 + 9 × 0.5 + 1（截斷）≈ 57.6 ULP。

— 停頓 —

上界 vs 實測。 57 ULP 是理論上界：假設每一步都碰到最壞情況。但如前述，B10 最壞和 B12 最壞不能同時發生（反相關）。實測 4,096 組隨機參數，E2E 最大誤差只有 6.04 ULP — 比上界寬鬆了 ~10 倍。這個差距正是「反相關效應」的量化證據。

結論：BS Delta 管線的誤差不是靠運氣控制的 — 除法放大和 erf 壓縮在數學上互相制衡，而 Φ 的 0.5 和 discount ≤1 提供額外的保險。整個 17 步管線的理論上界穩穩落在 ~57 ULP，實測遠低於此。

7Naive Lookup Table vs 多項式方法：精度等價成本

本系統的 E2E 精度為 ~2×10−14（6.04 ULP，Section 1）。如果不使用 Minimax 多項式和 Method B，而是以最直接的分段線性插值查表（piecewise linear interpolation）達到同樣的精度，需要多大的 lookup table？對整個 Proof #1 電路又有什麼影響？

精度匹配原則

為何不能直接用 E2E 誤差 2×10−14 作為每個函數的精度目標？
因為誤差會在 17 步管線中傳播與放大（Section 5：B10 除法放大 ~20×）。無論用多項式或查表，管線結構不變、放大倍率不變。因此，naive lookup 必須匹配我們每個函數的 kernel 精度，才能達到相同的 E2E 結果。

各函數的 kernel 精度目標（取自 Section 4 degree sweep 最佳值）：

理論基礎

分段線性插值在等距 N 分割下的最大誤差上界：

Emax ≤ h2 / 8 × max|f″(x)|    where h = (b − a) / N

解 Emax ≤ ε 得所需分割數：
N ≥ (b − a) × √(max|f″| / (8ε))

整體 Proof #1 電路影響

Naive lookup table 是電路全域共用的：無論幾個持倉，表只載入一次。但 ~3,500 萬筆 table 會直接決定電路 k 的下界，連帶拖慢整個證明（包括不受影響的 Merkle 和投組彙總部分）。

§ 20-position portfolio：當前 Delta 管線 ~290 rows/position × 20 = 5,800 rows，佔 44,615 total 的 13%。Naive 方案的 ~3,500 萬筆 lookup table 佔 35,021,770 total 的 99.9%。
∗ k = circuit size parameter, 2k = maximum rows。PLONK/KZG proof size 恆定（不隨 k 變化），但 proving time 隨 2k 線性增長。

Naive Lookup 的根本瓶頸：分段線性插值的收斂速度是 O(h2)，每多一位元精度需要表格大小 ×1.41。要達到 ~10−14 精度，表格膨脹到 ~3,500 萬筆。相比之下，d=12 Minimax 多項式的收斂速度是 O(h13)，每增加一個分段的精度增益是指數級的 — 同樣精度只需 ~2,000 筆參數。
結論：多項式方法以 ~2,000 筆參數實現 E2E ~2×10−14 精度，Delta 管線僅佔 Proof #1 電路的 13%。若改用 naive lookup table 達到同樣的 per-kernel 精度，需要 ~3,500 萬筆 table entries — 整個電路的 k 從 16 膨脹到 26（20-position portfolio），proving time 增加 ~1,024 倍。原本佔 13% 的 Delta 核心膨脹到佔比 99.9%，完全逆轉 Proof #1 的成本結構。

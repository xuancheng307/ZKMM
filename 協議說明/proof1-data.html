2Taylor vs Minimax Kernel 精度比較

在 kernel 層級（單一函數、無端對端傳播）比較 Taylor 展開與 Minimax（Remez）逼近的精度差距。

函數 pp d Taylor ULP Minimax ULP Minimax 優勢 有效位元 (Taylor) 有效位元 (Minimax)
ln 32 10 2.04×108 10.3 19,774,425× 4.4 bits 28.6 bits
ln 48 14 9.69×1012 142.5 68,043,806,829× 4.9 bits 40.8 bits
erf 32 10 3.21×106 7.5 428,698× 10.4 bits 29.1 bits
erf 48 14 3.78×109 14.4 263,317,498× 16.2 bits 44.2 bits
Taylor 的真正限制 — 精度，非係數符號。負係數本身不構成障礙：本系統的 Per-step Horner 使用 Euclidean 除法（rem_euclid）確保每步餘數 ∈ [0, S)，負係數透過 p − |c| 映射至 BN254 質數域，域運算自動正確。實際上，我們的 Minimax 係數也含負值（如 ln pp=48 init = −5.97×1012，addends 正負交替）。

Taylor 不適用的根本原因是精度：Taylor 展開最小化展開點處的導數，而非整個區間的最大誤差。在定點量化下，Taylor 的 worst-case ULP 比 Minimax 高 2.6×108 至 6.8×1010 倍（見上表）。此外，Taylor 在離開展開點後誤差快速發散，高次項的量化截斷進一步放大偏差。即使在 Taylor 收斂最好的 exp 函數（係數全正 1/k!），精度仍遠不及 Minimax。

5誤差傳播路徑（17 步 Pipeline）

BS Delta 管線有 4 條輸入分支，在 B7、B10、B16 三處匯合。不同分支的誤差獨立累積，只在匯合點合併。

4 條獨立分支：
  • A. S, K → B1-B2 (ratio, ln) → 匯入 B7
  • B. r, q, σ, T → B3-B6 (drift terms) → 匯入 B7
  • C. σ, T → B8-B9 (σ√T) → 匯入 B10
  • D. q, T → B14-B15 (exp(−qT)) → 匯入 B16

主幹：B7(加) → B10(除 — 放大器) → B11-B13(壓縮) → B16(乘 — 與分支 D 合併) → Δ

如何讀下表

  • 本步 δ — 這一步「自己製造」的新誤差（截斷誤差 + kernel 近似誤差）。
  • 累積上界 — 該中間值的總誤差帳戶餘額：先前累積的誤差，被本步放大或壓縮後，再加上本步 δ。
公式：累積上界 = 放大係數 × 輸入累積上界 + δ
放大係數 > 1（如 B10 除法 ×20）→ 累積暴增  |  < 1（如 B13 ×0.5）→ 累積壓縮

pp=24 相同結構。差異：B12 erf kernel δ=4.6（pp=48: 1.6）、B14-B15 Method B 3-byte δ=5（pp=48: 9）。最終上界 57.0 vs 57.6 ULP。

為什麼累積上界（57.6）≠ Σδ（~22）？
因為 B10 除法不只貢獻 δ=1 — 它把之前累積的 6.24 ULP 放大了 ~20 倍（除以小的 σ√T）。額外的 ~119 ULP 經三道壓縮後，最終多貢獻了 ~36 ULP。
57.6 ≈ 22（各步自製）+ 36（B10 放大殘留）
為什麼 B14–B15 只有 9 ULP？
因為 e−qT 是完全獨立的分支 D：從 q, T 直接計算，不經過 d1。Method B 的 6-byte 查表鏈 ~8 ULP + qT 乘法 1 ULP = 9 ULP。不受 B10 那 126 ULP 的影響。

放大與煞車：126 → 52 的壓縮路徑

B10 之後，三道煞車的淨壓縮率 = 0.707 × 1.13 × 0.50 ≈ 0.40：

6.24 ULP 進入 B10
  → B10 ÷(σ√T)：×20 → 125.84 ULP
  → B11 ÷√2：×0.707 → 89.98 ULP
  → B12 erf：×1.13 → 103.12 ULP
  → B13 Φ：×0.50 → 52.06 ULP

B16 合併：|exp| × εΦ + |Φ| × εexp + δ = 1 × 52.06 + 0.5 × 9.00 + 1.00 = 57.56 ULP

上界 vs 實測

57.6 ULP 是最壞情況上界（假設每步同時碰到最壞輸入）。但 B10 和 B12 的最壞情況互斥：

  • B10 最壞（×20 放大）→ 小 σ√T → 大 |d1| → erf 平坦 → B12 強力壓縮
  • B12 最壞（×1.13）→ d1 ≈ 0 → σ√T 正常 → B10 放大倍數低
實測驗證：4,096 組隨機參數，E2E 最大誤差 = 6.04 ULP — 僅為理論上界的 1/10。這個差距是「B10-B12 反相關效應」的量化證據。

最常見的誤解：「每一步各自製造一些誤差，全部加起來就是最終誤差。」如果這樣理解，δ 欄加起來約 22 ULP — 那最終怎麼是 57？多出的 35 從哪來？

— 停頓 —

答案：誤差不只「加總」，還會被「放大」。

想像管線是一條水管，每個閥門（步驟）會往水裡加一點染料（δ）。如果水管口徑不變，最終染料濃度就是所有閥門加的總和 ~22 ULP。但 B10 是一個瓶頸口 — 水管突然變窄 20 倍。之前加進去的所有染料被壓進更小的截面，濃度暴增。之前累積的 6.24 ULP 在 B10 變成 6.24 × 20 ≈ 125 ULP。（指向表中 B10 紅色列）

— 停頓 —

那為什麼 125 又降回 52？

因為後面有三個緩衝段，把管道拓寬了回去。（指向壓縮路徑圖）

B11 ÷√2：管道拓寬 ×0.707。因為只是做常數縮放。

B12 erf：S 型函數，導數最大才 1.13。不管輸入多大，erf 只微幅放大（d1≈0 時）或強力壓縮（|d1| 大時）。

B13 Φ = 0.5 + 0.5·erf：那個 0.5 直接砍半。

淨壓縮：0.707 × 1.13 × 0.50 ≈ 0.40。所以 126 × 0.40 + δ ≈ 52 ULP。

— 停頓 —

B14-B15 為什麼只有 9 ULP？

因為 exp(−qT) 是一條完全獨立的支線（分支 D）。它從 q 和 T 直接計算，跟 d1 毫無關係。Method B 用 6 個 byte 查表做乘法鏈，累積 ~8 ULP kernel 誤差 + qT 乘法的 1 ULP = 9 ULP。它根本不在 B10 那條管線上，所以不會繼承 126 ULP。（指向分支結構圖紫色路徑）

— 停頓 —

最後合併：B16。

Δ = exp(−qT) × Φ(d1)。主幹帶著 52 ULP，分支 D 帶著 9 ULP。乘法的誤差公式：|exp| × 52 + |Φ| × 9 + 1 = 1 × 52 + 0.5 × 9 + 1 = 57.6 ULP。exp ≤ 1 不放大 Φ；Φ 典型值 0.5 只放大 exp 一半。

— 停頓 —

為什麼實測只有 6 ULP？

因為 B10 和 B12 的最壞情況數學上互斥。B10 放大 20 倍需要 σ√T 很小 → |d1| 大 → erf' 趨近 0 → B12 幾乎不傳播誤差。反過來，B12 ×1.13 需要 d1≈0 → σ√T 正常 → B10 放大倍數遠小於 20。57 是「假設兩個最壞同時發生」— 但在 BS 公式中，這不可能。

總結：57.6 = 22（各步 δ）+ 36（B10 放大殘留）。22 是「自製誤差」，36 是「被放大再壓回來的殘留」。實測 6 ULP 遠低於理論上界，因為 BS 公式的結構讓最壞情況不會同時發生。

7Naive Lookup Table vs 多項式方法：精度等價成本

本系統的 E2E 精度為 ~2×10−14（6.04 ULP，Section 1）。如果不使用 Minimax 多項式和 Method B，而是以最直接的分段線性插值查表（piecewise linear interpolation）達到同樣的精度，需要多大的 lookup table？對整個 Proof #1 電路又有什麼影響？

精度匹配原則

為何不能直接用 E2E 誤差 2×10−14 作為每個函數的精度目標？
因為誤差會在 17 步管線中傳播與放大（Section 5：B10 除法放大 ~20×）。無論用多項式或查表，管線結構不變、放大倍率不變。因此，naive lookup 必須匹配我們每個函數的 kernel 精度，才能達到相同的 E2E 結果。

各函數的 kernel 精度目標（取自 Section 4 degree sweep 最佳值）：

理論基礎

分段線性插值在等距 N 分割下的最大誤差上界：

Emax ≤ h2 / 8 × max|f″(x)|    where h = (b − a) / N

解 Emax ≤ ε 得所需分割數：
N ≥ (b − a) × √(max|f″| / (8ε))

整體 Proof #1 電路影響

Naive lookup table 是電路全域共用的：無論幾個持倉，表只載入一次。但 ~3,500 萬筆 table 會直接決定電路 k 的下界，連帶拖慢整個證明（包括不受影響的 Merkle 和投組彙總部分）。

§ 20-position portfolio：當前 Delta 管線 ~290 rows/position × 20 = 5,800 rows，佔 44,615 total 的 13%。Naive 方案的 ~3,500 萬筆 lookup table 佔 35,021,770 total 的 99.9%。
∗ k = circuit size parameter, 2k = maximum rows。PLONK/KZG proof size 恆定（不隨 k 變化），但 proving time 隨 2k 線性增長。

Naive Lookup 的根本瓶頸：分段線性插值的收斂速度是 O(h2)，每多一位元精度需要表格大小 ×1.41。要達到 ~10−14 精度，表格膨脹到 ~3,500 萬筆。相比之下，d=12 Minimax 多項式的收斂速度是 O(h13)，每增加一個分段的精度增益是指數級的 — 同樣精度只需 ~2,000 筆參數。
結論：多項式方法以 ~2,000 筆參數實現 E2E ~2×10−14 精度，Delta 管線僅佔 Proof #1 電路的 13%。若改用 naive lookup table 達到同樣的 per-kernel 精度，需要 ~3,500 萬筆 table entries — 整個電路的 k 從 16 膨脹到 26（20-position portfolio），proving time 增加 ~1,024 倍。原本佔 13% 的 Delta 核心膨脹到佔比 99.9%，完全逆轉 Proof #1 的成本結構。

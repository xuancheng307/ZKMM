1問題與動機

傳統方式

無論是監管機構要求合規報告，或投資人要求基金證明風險管理 — 傳統方式都需要基金完整披露持倉。交易策略、標的選擇、部位大小全部暴露，造成不可逆的競爭劣勢。

結果：隱私 vs 透明不可兼得

零知識合規（本協議）

基金生成密碼學證明。監管機構或投資人驗證 proof 即可確認合規 — 不看到任何持倉資訊。proof 的存在本身就是合規的數學證據，任何人都可以獨立驗證。

結果：隱私與透明同時達成

合規公式

|Σi wi × Δi| ≤ τ
w = 持倉數量（正=做多，負=做空） | Δ = Black-Scholes Delta | τ = 合規門檻

選擇權的 Delta 由 Black-Scholes 公式計算，涉及三個超越函數： ln（自然對數）、erf（誤差函數）、exp（指數函數）。 這些函數在 ZK 電路中不存在 — 電路只有加法和乘法。 本論文的核心貢獻是用 Minimax 多項式逼近 + Horner 求值法 在電路內重建這三個函數。 完整公式見 Proof #1 技術細節 §0。

我們先理解這個協議要解決的核心問題。

對沖基金面臨一個根本矛盾：透明度與競爭力不可兼得。監管機構需要確認基金的風險在門檻之內，投資人需要確認自己的錢被合規管理 — 但傳統做法是完整披露持倉，這等於公開交易策略，對量化基金來說是致命的。

零知識證明打破這個僵局。基金在本地生成一個密碼學證明 — 這個 proof 的數學性質保證：如果 proof 驗證通過，則合規陳述為真。不需要信任基金，不需要看到持倉，不需要第三方。任何人 — 監管機構、投資人、甚至公眾 — 只要拿到 proof 和公開輸入，就能在毫秒內獨立驗證。

— 指向合規公式 —

合規檢查的核心是這個公式：所有持倉的加權 Delta 總和的絕對值不超過門檻 τ。Delta 來自 Black-Scholes 公式，需要 ln、erf、exp 三個超越函數 — 這些在零知識電路中不存在，是本論文的核心工程挑戰。

最上面四個角色：基金是唯一的 prover，負責生成證明。交易所簽署每筆交易。投資人和監管機構都是 verifier — 他們的驗證操作完全相同，只需要 proof 加公開輸入，毫秒內完成。差別在於監管機構額外擁有隨機抽查權，投資人則是自願驗證。

— 指向四棵樹 —

中間是四棵 Merkle 樹。Position State Tree 存所有持倉，Trade Action Tree 存已簽名交易，Market Data Tree 存市場公開數據，Event Tree 是新增的 — 處理所有非交易的狀態變更，例如股票拆股、特別現金股利。每棵樹計算出一個 Poseidon Merkle root，這四個 root 就是兩個 proof 的公開輸入。

底部兩個 ZK 證明獨立生成、獨立驗證。Proof #1 證明 Delta 合規（數學計算），Proof #2 證明狀態轉移合法（交易真實性 + 公司行動）。驗證者只需檢查三件事：兩個 proof 都通過，state_root 一致。就這樣，不需要看到任何持倉資訊。

驗證的開放性：ZK proof 的驗證不需要任何秘密 — 只需要 proof 檔案、公開輸入、和驗證金鑰（VK）。因此，驗證者不限於監管機構：基金的投資人（LP）可以自行下載 proof 並獨立驗證，無需信任任何中間方。proof 通過 = 數學證明合規，與誰驗證無關。
隱私分離原則：沒有任何一方能看到基金的完整持倉。基金知道自己的全貌，但只提交加密承諾。交易所各自只看到自己執行的部分交易。驗證者（監管機構和投資人）只看到 Merkle roots（幾個 hash 值）和兩個 proof。

協議涉及四方角色。基金是唯一的 prover，也是計算最密集的一方 — 每日需 5-10 分鐘生成兩個 proof。

交易所只負責簽名 — 每筆交易用 EdDSA 簽章。各所只看到自己執行的部分，沒有任何交易所能看到基金的完整持倉。

最關鍵的設計：驗證是完全開放的。ZK proof 的驗證只需要公開資訊 — proof 檔案、公開輸入、驗證金鑰。不需要私鑰、不需要權限、不需要專業知識。所以投資人可以自己驗證，和監管機構做一模一樣的操作。差別在於監管機構額外有「隨機抽查」的權力 — 可以要求基金開示特定持倉來交叉驗證。投資人沒有這個權力，但 proof 驗證本身已經提供了足夠的合規保證。

4四層信任模型

ZK proof 保證「給定輸入，計算正確」，但無法保證「輸入本身是真的」。 協議的 input authenticity（輸入真實性）由四層機制共同保障，每層針對不同的攻擊面：

L4

Random Audit 隨機抽查

針對密碼學無法防禦的場景：交易所用合法私鑰簽署虛假交易時，EdDSA 驗證仍會通過。 監管機構可要求基金開示特定持倉，與交易所交叉比對。

防禦目標：簽名者本身串通作弊（密碼學無法區分合法簽名+假內容 vs 合法簽名+真內容）
L3

ZK Computation 零知識計算

兩個 ZK 證明保證所有邏輯正確：Delta 加權合規（Proof #1）、狀態轉移合法（Proof #2）。 Proof 通過 ⇔ 陳述為真（基於 DLOG 與 Poseidon collision resistance 的數學保證）。

Proof #1 (Delta Compliance) + Proof #2 (State Transition)
L2

Cryptographic Authentication 密碼學認證

L2a 交易認證：每筆交易由執行交易所以 EdDSA 簽名，電路內驗證。偽造簽名在計算上不可行。
L2b 市場數據承諾：S, r, q, σ 經 Poseidon Merkle 樹承諾為 market_data_root，公開可驗證。
L2c 公司行動認證：拆股、現金股利等事件經 Event Tree 承諾為 event_root，以公開資訊為信任基礎（比 EdDSA 更強）。

L2a: BabyJubjub EdDSA | L2b: Market Poseidon Merkle | L2c: Event Poseidon Merkle
L1

Genesis Anchor 起點錨定

監管機構簽署初始狀態：State(0) = 空樹 root。Proof #2 的 state_root_prev 鏈從此起始， 確保狀態鏈有不可偽造的起點。這是協議中唯一的一次性信任假設。

Genesis commitment — 監管機構簽署的一次性信任假設

圖 2：Input authenticity 四層保障。L1 錨定狀態鏈起點，L2 認證每筆輸入（含公司行動），L3 證明計算正確，L4 補足密碼學盲區。

ZK proof 保證計算正確，但有一個容易被忽略的前提：proof 無法保證輸入本身是真的。如果基金填入假的市場數據，proof 照樣會通過 — 只是基於假數據的「正確計算」。

所以協議需要四層防禦。（從下往上指）

L1 是起點 — 監管機構簽署初始空狀態，確保狀態鏈的起點是可信的。L2 是密碼學認證 — 交易用 EdDSA 簽名（交易所私鑰），市場數據用 Merkle 承諾（公開可驗），公司行動用 Event Tree（公開資訊承諾，比 EdDSA 更強）。L3 是 ZK 計算本身。L4 是隨機抽查 — 防禦「合法簽名 + 假內容」的攻擊。

四層各自獨立，攻擊者必須同時突破所有層才能偽造合規。即使其中一層被破解（例如交易所串通），其他層仍然提供保護。

Event Tree 是新增的。為什麼需要它？考慮拆股 — Apple 做 4:1 拆股，你的 100 股變 400 股，這不是交易所的交易，交易所不會簽名。但持倉狀態確實改變了。過去我們沒有處理這類事件的機制。Event Tree 用公開資訊（公司公告）作為信任來源，比 EdDSA 更強 — 任何人都可以驗證 Apple 確實做了拆股，不需要依賴單一交易所的私鑰。

無需遞歸證明：兩個 proof 獨立生成、獨立驗證。驗證者只需檢查：(1) Proof #1 通過, (2) Proof #2 通過, (3) 兩者的 state_root 值一致。三個條件全部滿足 = 合規。監管機構和投資人執行完全相同的驗證邏輯。

兩個 proof 各自負責不同面向。（指向左欄）

Proof #1 是純數學 — 讀取持倉和市場數據，計算 Black-Scholes Delta，驗證加權總和 ≤ τ。核心挑戰是三個超越函數 ln, erf, exp 的電路實作。公開輸入 4 個：state_root, market_data_root, τ, today。

— 指向右欄 —

Proof #2 是狀態管理 — 驗證每筆交易的 EdDSA 簽名，套用到持倉狀態樹，同時處理公司行動（Event Tree Merkle proof + 比率約束）。公開輸入4 個：state_root_prev, state_root, batch_root, event_root。event_root 是新增的 — 承諾了這個結算期所有的公司行動。

兩個 proof 透過 state_root 銜接：Proof #2 輸出的 state_root 必須等於 Proof #1 讀取的 state_root。驗證者獨立驗證兩個 proof，然後比對 state_root 值 — 三件事都通過就是合規。不需要遞歸證明，設計簡潔、高效。

這張矩陣列出了所有我們考慮過的攻擊向量。（逐行指）

前兩個是數據偽造 — 偽造 σ 和市場數據。防禦方式不同：σ 用正向驗證（計算 BS price 是否接近結算價），市場數據用 Merkle 路徑綁定。

隱藏持倉是最危險的攻擊 — 基金跳過高風險持倉不包含在 proof 中。防禦靠 Proof #2 的完整性保證：state_root 承諾了所有 128 個 slots，包括 dummy。

偽造公司行動是新增的攻擊向量。基金可能捏造一個不存在的 2:1 拆股來把持倉數量翻倍。Event Tree 的防禦：event_root 必須正確 — 它承諾的是公開可驗證的公司公告。任何人都可以獨立建構 Event Tree 來驗證 event_root 是否正確。

最後，交易所串通是密碼學無法防禦的 — 合法私鑰簽署假交易，簽名驗證會通過。這就是為什麼需要 L4 隨機抽查作為最後一道防線。

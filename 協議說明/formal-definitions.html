2有限域

Definition 2.1 — BN254 素數域 𝔽p

令 p = 21888242871839275222246405745257275088548364400416034343698204186575808495617， p 為 254-bit 素數。素數域 𝔽p = {0, 1, …, p − 1}，配備模 p 的加法與乘法。

Definition 2.2 — 域擴張 𝔽p2

𝔽p2 = 𝔽p[u] / (u2 + 1)，元素表為 a + bu，a, b ∈ 𝔽p。用於 BN254 的 G2 定義。

Definition 2.3 — 乘法子群 ℍ

取最大 k 使得 2k | (p − 1)。令 ω 為 𝔽p 中 2k 次本原單位根。 對任意電路大小 n = 2s (s ≤ k)，evaluation domain ℍ = {1, ω, ω2, …, ωn−1}。

BN254 的 k = 28，故最大電路大小 n = 228 = 268,435,456 行。

3橢圓曲線

Definition 3.1 — BN254 曲線

E(𝔽p) : y2 = x3 + 3，群階 |E(𝔽p)| = r，其中

r = 21888242871839275222246405745257275088548364400416034343698204186575808495617

注意：BN254 的 r = p（self-pairing 特性），scalar field 與 base field 同構。

G1 = E(𝔽p)[r]（r-torsion subgroup over 𝔽p）

G2 = E'(𝔽p2)[r]（twisted curve over 域擴張）

GT ⊂ 𝔽p12*（目標群，r 階子群）

Definition 3.2 — Bilinear Pairing

最優 Ate pairing e : G1 × G2 → GT，滿足：

  1. Bilinearity：e(aP, bQ) = e(P, Q)ab，∀ a, b ∈ 𝔽r
  2. Non-degeneracy：e(G1, G2) ≠ 1GT（G1, G2 為生成元時）
  3. Efficient computability：存在 PPT 演算法計算 e
Definition 3.3 — BabyJubjub 曲線

Twisted Edwards curve over 𝔽p（BN254 scalar field）：

ax2 + y2 = 1 + dx2y2

其中 a = 168700, d = 168696。群階 = 8 × r'（cofactor 8），r' 為大素數。

用途：EdDSA 簽章。選用原因：定義在 BN254 scalar field 上，可直接在 halo2 電路中做群運算（無需非原生域算術）。

4定點數算術

Definition 4.1 — 定點數表示 FPpp

給定精度參數 pp ∈ ℤ+，實數 x ∈ ℝ 的定點數表示為：

fppp(x) = ⌊x · 2pp⌉ ∈ ℤ

逆映射（解碼）：x ≈ fppp(x) / 2pp。表示誤差 |x − fppp(x)/2pp| ≤ 2−(pp+1)。

在電路中，fppp(x) 嵌入 𝔽p（要求 fppp(x) < p）。負數以 p − |fppp(x)| 表示。

Definition 4.2 — 定點乘法 fpmul

對 a, b ∈ FPpp，定點乘法定義為：

fpmul(a, b) = ⌊a · b / 2pp⌋

在電路中實現為：計算 a · b（得到 2pp-bit 中間值），然後右移 pp 位（截斷低 pp 位）。

截斷約束：a · b = q · 2pp + rem，其中 0 ≤ rem < 2pp，輸出 q。

Definition 4.3 — 定點除法 fpdiv

對 a, b ∈ FPpp（b ≠ 0），定點除法定義為：

fpdiv(a, b) = ⌊a · 2pp / b⌋

在電路中實現為：先左移 a（乘 2pp），再整除 b。約束：a · 2pp = q · b + rem，0 ≤ rem < b。

Definition 4.4 — ULP（Unit in the Last Place）

對精確值 xexact 與電路計算值 xcircuit，ULP 誤差定義為：

ULP(x) = |fppp(xexact) − xcircuit|

1 ULP = 2−pp 的絕對誤差。ULP 是無量綱整數，量測定點數的最小分辨單位誤差。

Definition 4.5 — 有效位數（Effective Bits）

給定最大 ULP 誤差 ULPmax，有效位數定義為：

beff = pp − log2(ULPmax)

例：pp = 32, ULPmax = 8 ⇒ beff = 32 − 3 = 29 有效位。

Lemma 4.1 — 單次截斷誤差界

每次 fpmul 或 fpdiv 操作引入的截斷誤差滿足 |εtrunc| ≤ 1 ULP。

Proof sketch：floor 除法丟棄餘數 rem ∈ [0, 2pp)，最大損失 = 2pp − 1 < 2pp = 1 ULP（以原尺度計）。▮

Lemma 4.2 — Horner d 步累積截斷界

Horner 求值法對 d 次多項式執行 d 次 fpmul + d 次加法。Per-step truncation 下：

ULPtotal ≤ Σi=1..d |ci| / 2pp + d

其中第一項為係數量化誤差的放大，第二項為 d 次截斷的累積。實務中由第二項主導，故 ULPtotal ≈ d/2（期望值）。

PART II 密碼學原語

5Poseidon 雜湊

Definition 5.1 — Poseidon 置換

Poseidon 置換 π : 𝔽pt → 𝔽pt 由 R = rf + rp 輪組成，每輪依序執行：

  1. AddRoundConstants (ARC)：xi ← xi + cr,i
  2. S-box：xi ← xiα，其中 α = 5
     Full round（前 rf/2 輪 + 後 rf/2 輪）：所有 t 個元素套用 S-box
     Partial round（中間 rp 輪）：僅第一個元素套用 S-box
  3. MDS mixing：x ← M · x，M ∈ 𝔽pt×t 為 MDS 矩陣

BN254 參數：t = 3（2-to-1 hash），rf = 8, rp = 57，α = 5。

Definition 5.2 — Poseidon 海綿雜湊

HPoseidon : 𝔽p* → 𝔽p，使用海綿結構（sponge construction）：

  1. 初始化狀態 s = (0, 0, …, 0) ∈ 𝔽pt
  2. Absorb：將輸入分為 rate 大小的區塊，每區塊加入狀態前 r 個元素後執行 π
  3. Squeeze：輸出狀態第一個元素作為雜湊值

rate = t − 1 = 2，capacity = 1。本協議中，Merkle 雜湊取 H(left, right) = Poseidon(left, right)。

Proposition 5.1 — Poseidon 安全性

在 algebraic attack model 下，Poseidon 的碰撞抵抗（collision resistance）與原像抵抗（preimage resistance）由輪數保證： rf + rp 需超過 Gröbner basis 攻擊與差分/線性攻擊的安全邊界。BN254 參數提供 128-bit 安全等級。

選用 Poseidon 而非 SHA-256 的原因：Poseidon 以 𝔽p 原生運算定義，電路成本約 ~330 constraints vs SHA-256 的 ~25,000 constraints。

6Merkle 樹

Definition 6.1 — Poseidon Merkle 樹

深度 d 的 Poseidon Merkle 樹 T(d, H) 定義為完整二元樹：

  • 葉節點：Li = HPoseidon(field0, field1, …, fieldm−1)，i ∈ [0, 2d)
  • 內部節點：Nj = HPoseidon(Nleft(j), Nright(j))
  • Root：root = N0（頂層節點）

空葉以全零欄位表示：Lempty = HPoseidon(0, 0, …, 0)。

Definition 6.2 — Merkle Membership Proof

對葉 Li，membership proof 為 πMerkle = (s0, s1, …, sd−1, b)，其中：

  • sj ∈ 𝔽p：第 j 層的 sibling hash
  • b ∈ {0, 1}d：index 的二進位表示（決定左/右）
Definition 6.3 — Merkle 驗證函數

MerkleVerify(root, leaf, idx, π) → {0, 1}：

cur ← leaf
for j = 0 to d − 1:
  if b[j] = 0: cur ← H(cur, s[j])
  else: cur ← H(s[j], cur)
return (cur == root)

7EdDSA 簽章

Definition 7.1 — EdDSA 金鑰生成

KeyGen() → (sk, pk)：

  1. 隨機取 sk ∈$ 𝔽r'
  2. pk = sk · B，B 為 BabyJubjub 的基點
Definition 7.2 — EdDSA 簽名

Sign(sk, m) → (R, s)：

  1. r = HPoseidon(sk, m)（確定性 nonce）
  2. R = r · B
  3. h = HPoseidon(R.x, R.y, pk.x, pk.y, m)
  4. s = r + h · sk mod r'
Definition 7.3 — EdDSA 驗簽

Verify(pk, m, R, s) → {0, 1}：

  1. h = HPoseidon(R.x, R.y, pk.x, pk.y, m)
  2. 檢查 s · B == R + h · pk（橢圓曲線點等式）

電路內驗簽：Step 2 的點等式展開為 𝔽p 上的加法與乘法約束（約 ~1,500 constraints）。

8KZG 多項式承諾

Definition 8.1 — Structured Reference String (SRS)

給定最大多項式次數 dmax，可信設置產生：

SRS = ([1]1, [τ]1, [τ2]1, …, [τdmax]1,   [1]2, [τ]2)

其中 τ ∈$ 𝔽r 為 toxic waste（設置完成後必須銷毀）。

本協議使用 Hermez/Zcash Powers of Tau ceremony 產生的 SRS。

Definition 8.2 — KZG Commit

對多項式 f(X) = Σi=0..d ci Xi ∈ 𝔽r[X]：

Commit(f, SRS) = Σi=0..d ci [τi]1 = [f(τ)]1 ∈ 𝔾1

承諾為單個群元素（48 bytes compressed）。

Definition 8.3 — KZG Open（Evaluation Proof）

證明 f(z) = v：

  1. 計算商多項式 q(X) = (f(X) − v) / (X − z)
  2. proof π = [q(τ)]1 = Commit(q, SRS)
  3. 輸出 (v, π)

q(X) 整除的充要條件恰好是 f(z) = v（多項式餘式定理）。

Definition 8.4 — KZG Verify

Verify(C, z, v, π) → {0, 1}：

e(C − [v]1,   [1]2)   ≟   e(π,   [τ − z]2)

等價於檢查 f(τ) − v = q(τ) · (τ − z)，利用 pairing 在指數層驗證。

驗證成本：2 pairings（恆定，與多項式次數無關）。

Definition 8.5 — SHPLONK 批次開啟

SHPLONK（Simplified HPLONK）將多個多項式在多個點的 evaluation proof 聚合為單一 proof：

給定 {(fi, zi, vi)}i=1..m，verifier 選隨機挑戰 γ，聚合為：

h(X) = Σi γi · (fi(X) − vi) / (X − zi)

proof = [h(τ)]1，驗證為單一 pairing check。

本協議 halo2+KZG 使用 SHPLONK 作為最終 opening scheme。

Theorem 8.1 — KZG Binding

在 d-Strong Diffie-Hellman (d-SDH) 假設下，KZG 承諾滿足 evaluation binding：

對任何 PPT 對手 A，Pr[A 輸出 (C, z, v, π, v', π') 使得 v ≠ v' 且 Verify(C, z, v, π) = Verify(C, z, v', π') = 1] ≤ negl(λ)。

直覺：若 f(z) = v 且 f(z) = v'（v ≠ v'），則 f(X) − v 和 f(X) − v' 不可能同時被 (X − z) 整除。

Note：標準 KZG 不具備 hiding 性質（承諾 C 洩漏 f(τ)）。halo2 在 advice columns 加入 blinding rows（隨機列），使承諾統計上不可區分，從而達成 zero-knowledge。

PART III 證明系統

9Plonkish 算術化（halo2）

Definition 9.1 — Plonkish Arithmetization

A Plonkish arithmetization over 𝔽p is a tuple A = (n, ma, mf, mi, S, G, L, σ) where：

  • n：行數（rows），包含 usable rows + blinding rows
  • ma：advice column 數（private witness）
  • mf：fixed column 數（public constants, selectors）
  • mi：instance column 數（public inputs）
  • S = {sj : [n] → {0, 1}}：selector 多項式集合（啟用/禁用 gate）
  • G = {gk}：gate 多項式集合，每個 gk 定義一組多項式約束
  • L = {(Tl, inputl)}：lookup 引數集合（table + input expression）
  • σ：cell 置換（copy constraints over all columns）
Definition 9.2 — Satisfying Assignment

給定 A，一個 satisfying assignment 為 advice matrix W ∈ 𝔽pn × ma，滿足：

  1. Gate constraints：∀ k, ∀ i ∈ [n]: sk(i) · gk(Wi,·, Fi,·) = 0
  2. Lookup constraints：∀ (Tl, inputl): inputl(W, F) ∈ Tl  （on every active row）
  3. Copy constraints：∀ (c1, c2) ∈ σ: W[c1] = W[c2]
  4. Instance constraints：指定的 instance cells 與公開輸入值一致
Definition 9.3 — halo2 Relation
Rhalo2 = { (x, w) : x = instance columns, w = advice columns, (x, w) is a satisfying assignment of A }

x（public input）是 verifier 已知的；w（witness）是 prover 的私有輸入。

Definition 9.4 — halo2 Proof System

halo2 proof system 為三元組 Π = (Setup, Prove, Verify)：

  • Setup(A, SRS) → (pk, vk)
     pk 包含：所有 fixed column commitments, permutation commitments, lookup table commitments
     vk 包含：fixed column commitments（的子集）+ 電路結構雜湊
  • Prove(pk, x, w) → π
     Step 1: Commit advice columns（含 blinding rows）
     Step 2: Commit lookup permuted columns
     Step 3: Commit permutation argument / lookup argument / gate quotient
     Step 4: Evaluate all polynomials at challenge point ξ
     Step 5: SHPLONK opening proof
  • Verify(vk, x, π) → {0, 1}
     Step 1: 驗證 transcript consistency（Fiat-Shamir challenges）
     Step 2: 驗證 gate/lookup/permutation 商多項式在 ξ 處的值
     Step 3: SHPLONK pairing check
Definition 9.5 — Custom Gate

一個 custom gate 是一個多變量多項式 g : 𝔽pm → 𝔽p，作用於同一列的 相對行 cells。

例：本協議的 Horner gate（2 relative rows, 4 columns）：

ghorner(acur, bcur, anext, coeff) = acur · bcur + coeff − anext · 2pp − rem

selector shorner(i) = 1 僅在 Horner 步驟的行啟用。

Definition 9.6 — Lookup Argument

Lookup 引數證明一組 input expressions 的值都存在於預定義的 table 中：

{input(Wi,·, Fi,·) : i ∈ active rows} ⊆ Table

halo2 使用基於 permutation 的 lookup（類似 Plookup）。本協議使用的 table：

  • ByteTable[0..255]：range check（byte decomposition）
  • Pow2Table：2 的冪次（用於 bit shift）
  • Method B exp tables：預計算 ebyte·28k/S · S（per byte position）
Definition 9.7 — Copy Constraint（Permutation Argument）

置換引數 σ 定義了跨不同 columns/rows 的 cell 等式約束：

W[cola, rowi] = W[colb, rowj]

用途：將一個 gate 的輸出連接到另一個 gate 的輸入，構成計算 pipeline。

實現：grand product argument over σ-permuted column indices。

10零知識證明系統

Definition 10.1 — 非互動式論證系統

一個非互動式論證系統（Non-Interactive Argument）for relation R 為三元組 Π = (Setup, Prove, Verify)：

  • Setup(1λ, R) → (crs, pk, vk)：產生公共參數
  • Prove(pk, x, w) → π：給定 (x, w) ∈ R，產生證明
  • Verify(vk, x, π) → {0, 1}：驗證證明
Definition 10.2 — Completeness

若 (x, w) ∈ R，則誠實 prover 產生的 proof 一定被接受：

Pr[Verify(vk, x, Prove(pk, x, w)) = 1] = 1

（Perfect completeness：概率恰好為 1。）

Definition 10.3 — Knowledge Soundness

對任何 PPT prover P*，若 P* 能使 verifier 接受，則存在 PPT extractor E 使得：

Pr[Verify(vk, x, π) = 1 ∧ (x, w) ∉ R : π ← P*(pk, x), w ← E(pk, x, P*)] ≤ negl(λ)

直覺：不存在「知道 proof 但不知道 witness」的作弊者。Extractor 可以從作弊者的內部狀態提取出合法的 witness。

halo2+KZG 在 Algebraic Group Model (AGM) + Random Oracle (RO) 下滿足此性質。

Definition 10.4 — Zero-Knowledge

存在 PPT simulator S 使得：對任何 PPT distinguisher D，

|Pr[D(Prove(pk, x, w)) = 1] − Pr[D(S(pk, x)) = 1]| ≤ negl(λ)

直覺：真實 proof 與 simulator（不知道 w）產生的 proof 不可區分。Verifier 從 proof 中學不到任何關於 w 的資訊。

halo2 通過 blinding rows（advice columns 的最後若干行填入隨機值）實現統計零知識。

Theorem 10.1 — halo2+KZG 安全性

令 Π = halo2 with KZG+SHPLONK backend over BN254。在以下假設下：

  • d-SDH assumption holds in (G1, G2)
  • Algebraic Group Model (AGM)
  • Random Oracle Model (transcript hash modeled as RO)

Π 滿足 perfect completeness、computational knowledge soundness、statistical zero-knowledge。

Proof size = O(1) group elements（與電路大小無關）。Verify time = O(1) pairings + O(|x|) field ops。

PART IV 金融模型

11Black-Scholes 選擇權定價

Definition 11.1 — 幾何布朗運動（GBM）

標的資產價格 St 遵循：

dSt = (r − q) St dt + σ St dWt

其中 r = 無風險利率，q = 連續股息率，σ = 波動率，Wt = 標準 Wiener process。

Definition 11.2 — Black-Scholes 定價公式

歐式看漲選擇權價格：

C = S e−qT N(d1) − K e−rT N(d2)

歐式看跌選擇權價格（put-call parity）：

P = K e−rT N(−d2) − S e−qT N(−d1)

其中 S = 現貨價，K = 履約價，T = 到期時間（年），r, q, σ 如 Definition 11.1。

Definition 11.3 — d1, d2 參數
d1 = [ln(S/K) + (r − q + σ2/2) T] / (σ √T)
d2 = d1 − σ √T
Definition 11.4 — 標準常態分佈 CDF 與 erf 的關係

標準常態累積分佈函數 N(x) 定義為：

N(x) = ½ [1 + erf(x / √2)]

其中 erf 為誤差函數：

erf(x) = (2/√π) ∫0x e−t2 dt

本協議透過 Minimax 多項式逼近 erf 來實現 N(x)（見 Part V）。

12BS Greeks

Definition 12.1 — Delta (Δ)

選擇權價格對標的資產價格的一階偏導數：

Δcall = ∂C/∂S = e−qT N(d1)     Δput = ∂P/∂S = −e−qT N(−d1)

Δ ∈ [0, 1] for calls，Δ ∈ [−1, 0] for puts。

在電路中，Δ 以 FPpp 表示。計算 pipeline 共 17 步（見 Proof #1 §5）。

Definition 12.2 — Gamma (Γ)

Delta 對標的資產價格的一階偏導數（選擇權價格的二階偏導數）：

Γ = ∂Δ/∂S = e−qT φ(d1) / (S σ √T)

其中 φ(x) = (1/√(2π)) e−x2/2 為標準常態 PDF。

Γ ≥ 0（凸性）。計算需額外一次 exp 求值（e−d12/2），複用 d1 pipeline。

Definition 12.3 — Vega (V)

選擇權價格對波動率的偏導數：

V = ∂C/∂σ = S e−qT φ(d1) √T

與 Gamma 共用 φ(d1) 計算。

可擴展性：所有 BS Greeks 共用 d1 計算 pipeline（17 步中的 B1-B12）。不同 Greeks 僅在 B13 之後分叉。新增一個 Greek 的邊際電路成本約 10-30%（非 100%）。

13Merton 結構模型

Definition 13.1 — 違約距離（Distance to Default）
DD = [ln(V/D) + (μ − σV2/2) T] / (σV √T)

V = 公司資產價值，D = 債務面值，μ = 資產預期報酬率，σV = 資產波動率。

Definition 13.2 — 違約機率（Probability of Default）
PD = N(−DD) = ½ erfc(DD / √2)

利用 erf 的互補關係 erfc(x) = 1 − erf(x)。

電路中複用 ln + erf kernel（與 BS Delta 共用），僅參數不同。

14合規函數

Definition 14.1 — Portfolio Weighted Delta

給定 N 個持倉 {(wi, Δi)}i=1..N：

Δport = Σi=1N wi · Δi

wi ∈ ℤ（正=做多，負=做空），Δi ∈ FPpp。加法在 𝔽p 中精確（無截斷）。

Definition 14.2 — Delta 合規謂詞
φ(Δport, τ) := |Δport| ≤ τ

在電路中實現為：計算 |Δport|（取絕對值），然後 range check τ − |Δport| ≥ 0。

τ 為 instance column（public input），verifier 可見。

Definition 14.3 — 門檻語義

τ 以 Delta 單位表示（非名目金額）：

  • τ = 0.05 · 2pp：portfolio net Delta 不超過 5% of notional
  • τ = 0：完美 delta-neutral（實務中不可能精確達成）

換算為名目金額：|exposure| = |Δport| · S · multiplier，其中 S = 標的現價，multiplier = 合約乘數。

PART V 多項式逼近理論

15Minimax 逼近

Definition 15.1 — 最佳一致逼近（Best Uniform Approximation）

給定連續函數 f : [a, b] → ℝ 和多項式次數 d，最佳一致逼近為：

p* = arg minp ∈ Pd ‖f − p‖∞

其中 Pd 為次數 ≤ d 的多項式集合，‖g‖∞ = maxx ∈ [a,b] |g(x)|。

p* 稱為 Minimax 多項式，也稱 Chebyshev 最佳逼近。

Theorem 15.1 — Chebyshev Equioscillation Theorem

p* ∈ Pd 是 f 在 [a, b] 上的最佳一致逼近 若且唯若 誤差 e(x) = f(x) − p*(x) 在 [a, b] 上有至少 d + 2 個交替極值點 x0 < x1 < … < xd+1，使得：

e(xi) = (−1)i · ‖e‖∞   ∀ i

推論：

  • 存在性：p* 一定存在（Weierstrass 定理 + 緊集）
  • 唯一性：p* 是唯一的
  • 最優性特徵：任何其他 d 次多項式的最大誤差嚴格大於 ‖e‖∞
Definition 15.2 — Remez Exchange Algorithm

迭代求解 Minimax 多項式的標準演算法：

  1. Init：選取 d + 2 個初始參考點 {xi}0..d+1（通常取 Chebyshev 節點）
  2. Level：解 (d + 2) × (d + 2) 線性系統 p(xi) + (−1)i E = f(xi)，得多項式 p 和等振幅 E
  3. Exchange：找 |f(x) − p(x)| 的全域極值點，替換參考集中偏差最小的點
  4. Converge：重複 2-3 直到 E 收斂（通常 10-20 次迭代）

收斂後的 p 即為 Minimax 多項式 p*（在機器精度內）。

16Horner 求值法

Definition 16.1 — Horner 遞推（實數域）

多項式 p(x) = c0 + c1x + … + cdxd 的 Horner 求值：

h0 = cd
hk+1 = hk · x + cd−k−1   for k = 0, 1, …, d − 1
p(x) = hd

運算量：d 次乘法 + d 次加法（最優）。

Definition 16.2 — 定點 Horner（Per-step Truncation）

在 FPpp 中，每步乘法後截斷以防溢位：

h0 = cd
hk+1 = ⌊hk · x / 2pp⌋ + cd−k−1   for k = 0, …, d − 1
output = hd

每步中間值 ≤ 22pp+1（已截斷），永遠不超過 𝔽p 容量。

電路約束：hk · x = qk · 2pp + remk，remk ∈ [0, 2pp) 以 byte decomposition range check 保證。

Definition 16.3 — 定點 Horner（Deferred Truncation）

推遲截斷：所有乘法和加法在整數域完成，最後一次截斷：

h0 = cd
hk+1 = hk · x + cd−k−1 · 2k·pp   (no truncation)
output = ⌊hd / 2d·pp⌋

優點：只截斷一次（ULP ≤ 1）。缺點：中間值需 (d+1)·pp bits。

Theorem 16.1 — Deferred Truncation 溢位條件

Deferred Horner 可行的充要條件：中間值不超過 𝔽p 的表示範圍：

(d + 1) · pp < ⌈log2 p⌉ = 254   (for BN254)
pp最大 d（deferred）狀態
16d ≤ 14可行 — 所有實務需求 d ≤ 7
24d ≤ 9可行 — 本協議 d = 8
32d ≤ 6不足 — 需 d = 10（ln, erf）
48d ≤ 4不足 — 需 d = 14

結論：pp ≥ 32 時 deferred truncation 溢位，必須使用 per-step truncation。

Theorem 16.2 — Per-step 累積誤差界

Per-step Horner 的總 ULP 誤差由兩部分組成：

ULPtotal = ULPRemez + ULPtrunc
  • ULPRemez：Minimax 多項式逼近誤差（固定，由 Remez 保證）
  • ULPtrunc ≤ d（最壞情況），期望值 ≈ d/2

存在最優 d* 使得 ULPtotal 最小：d < d* 時 Remez 誤差主導，d > d* 時截斷主導。

17係數條件數

Definition 17.1 — 係數條件數（Coefficient Conditioning）

在相同 kernel 參數 (pp, d, nbytes) 下，不同函數因 Minimax 係數結構的差異，產生不同的定點數精度：

CR(f, g) = ULPmax(f) / ULPmax(g)

影響因子：

  • 有效 Horner 步數：奇函數（如 erf）半數係數為零，實際僅 ⌈d/2⌉ 步有截斷
  • 係數交替性：交替正負係數（如 ln）每步截斷誤差可能加強
  • 量化靈敏度：高 pp 時，定點表示的相對誤差更小，但累積步數更多
Observation 17.1 — Conditioning 隨 pp 放大

實驗觀測（同 d, 同 kernel 架構）：

ppln ULPmaxerf ULPmaxCR(ln/erf)
1611~1.0x

16. 金融模型背景

本論文的兩個應用情境分別使用 Black-Scholes 與 Merton 結構性模型。 本章介紹這些模型的數學基礎、所需的超越函數，以及它們為何需要零知識證明。

核心定義

Geometric Brownian Motion (GBM) 幾何布朗運動
描述資產價格隨機演化的連續時間隨機過程： $dS_t = \mu S_t \, dt + \sigma S_t \, dW_t$， 其中 $\mu$ 為漂移率、$\sigma$ 為波動率、$W_t$ 為 Wiener 過程。 Black-Scholes 與 Merton 模型都假設標的資產服從 GBM。
Option 選擇權
一種衍生性金融商品，賦予持有者在到期日以約定價格（履約價 $K$）買入（Call）或賣出（Put）標的資產的權利（非義務）。
Delta ($\Delta$) Delta 值
選擇權價格對標的資產價格的一階偏微分：$\Delta = \partial C / \partial S$。 衡量標的價格變動 1 元時，選擇權價格變動多少。Call 的 $\Delta \in [0, 1]$，Put 的 $\Delta \in [-1, 0]$。 投資組合的總 $\Delta$ 接近 0 表示「Delta 中性」——已做好對沖。
Credit Default Swap (CDS) 信用違約交換
一種信用衍生商品。買方定期支付保費（spread），若參考實體（reference entity）發生信用事件（違約）， 賣方賠償買方的損失。功能上等同於「公司債的保險」。
Probability of Default (PD) 違約機率
在特定時間範圍內，債務人無法履行債務的機率。 投資等級（investment grade）公司的年化 PD 通常 < 2%，投機等級（speculative grade）> 4%。 PD 的微小差異（2 個基點）可能影響合規判定。
Normal CDF $\Phi(x)$ 標準常態累積分布函數
$\Phi(x) = \frac{1}{2}\bigl[1 + \text{erf}(x / \sqrt{2})\bigr]$， 其中 $\text{erf}(z) = \frac{2}{\sqrt{\pi}} \int_0^z e^{-t^2} dt$。 Black-Scholes 與 Merton 的公式都透過 $\Phi$ 將 $d_1, d_2$ 轉換為機率。 在 ZK 電路中，$\Phi$ 的計算歸結為 erf 函數的多項式逼近。

Black-Scholes 選擇權定價模型

模型假設

Black-Scholes (1973) 假設：

  • 標的資產價格服從幾何布朗運動（GBM）
  • 無風險利率 $r$ 為常數
  • 波動率 $\sigma$ 為常數
  • 無交易成本、無套利機會
  • 歐式選擇權（只在到期日行使）

定價公式

歐式 Call 的價格：

$$C = S \, e^{-qT} \, \Phi(d_1) - K \, e^{-rT} \, \Phi(d_2)$$

其中：

$$d_1 = \frac{\ln(S/K) + (r - q + \sigma^2/2) \, T}{\sigma \sqrt{T}}, \quad d_2 = d_1 - \sigma \sqrt{T}$$
符號意義典型值
$S$標的資產現價100
$K$履約價（strike price）95 – 105
$T$到期時間（年）0.25 – 2
$r$無風險利率0.02 – 0.05
$q$股息殖利率0 – 0.03
$\sigma$波動率0.15 – 0.60

Delta：對沖的核心

對定價公式求 $\partial / \partial S$：

$$\Delta_{\text{call}} = e^{-qT} \, \Phi(d_1), \quad \Delta_{\text{put}} = e^{-qT} \bigl[\Phi(d_1) - 1\bigr]$$
直觀理解

Delta 是選擇權價格對股價的「敏感度」。$\Delta = 0.6$ 表示股價漲 1 元，選擇權漲約 0.6 元。 Delta 中性（$|\Delta_{\text{total}}| \approx 0$）是機構對沖的基本要求—— 組合價值不隨市場方向變動。

計算 Delta 需要的超越函數

計算步驟超越函數電路中的處理
$\ln(S/K)$ 自然對數 $\ln$ Minimax 多項式逼近 + Horner 求值
$\Phi(d_1) = \frac{1 + \text{erf}(d_1/\sqrt{2})}{2}$ 誤差函數 $\text{erf}$ Minimax 多項式逼近 + Horner 求值
$e^{-qT}$（股息折現因子） 指數函數 $\exp$ Minimax 多項式逼近 + Horner 求值

這三個超越函數（$\ln, \text{erf}, \exp$）正是本論文 ZK 電路必須處理的核心—— 它們無法在有限域上精確計算，必須透過多項式逼近來實現。

Merton 結構性信用模型

核心思想：股權即 Call Option

Merton (1974) 的關鍵洞察：將公司的股權視為一個以公司資產為標的、 以負債面值為履約價的 Call Option。

到期時 $V_T > D$（資產 > 負債）

  • 公司有剩餘價值 $V_T - D$
  • 股東拿走剩餘 → 股權有價值
  • 等同 Call Option 在價內 (ITM)
  • 不違約

到期時 $V_T < D$（資產 < 負債）

  • 公司資不抵債
  • 股東的有限責任 → 股權歸零
  • 等同 Call Option 在價外 (OTM)
  • 違約

違約機率公式

假設公司資產價值也服從 GBM：$dV_t = \mu V_t \, dt + \sigma_V V_t \, dW_t$

在風險中立測度下，違約機率（Probability of Default）為：

$$PD = \Phi(-d_2) = \frac{1 - \text{erf}\bigl(d_2 / \sqrt{2}\bigr)}{2}$$

其中：

$$d_2 = \frac{\ln(V/D) + (r - \sigma_V^2 / 2) \, T}{\sigma_V \sqrt{T}}$$
符號意義典型值
$V$公司資產市值不可直接觀察
$D$負債面值（違約門檻）短期負債 + 0.5 $\times$ 長期負債
$\sigma_V$資產波動率0.10 – 0.50
$r$無風險利率0.02 – 0.05
$T$評估期限通常 1 年
與 Black-Scholes 的數學同構

比較兩者的公式結構：

Black-Scholes DeltaMerton PD
計算目標$\Phi(d_1)$$\Phi(-d_2)$
ln 的自變量$\ln(S/K)$$\ln(V/D)$
漂移項$(r - q + \sigma^2/2) T$$(r - \sigma_V^2/2) T$
除數$\sigma \sqrt{T}$$\sigma_V \sqrt{T}$

兩者在數學結構上幾乎一致——都需要 $\ln$ 和 $\text{erf}$，差別只在語義（選擇權 Delta vs 違約機率） 和精度需求。

計算 PD 需要的超越函數

計算步驟超越函數備註
$\ln(V/D)$ 自然對數 $\ln$ 對精度要求最高——V/D 接近 1 時 ln 接近 0，微小誤差被放大
$\Phi(-d_2) = \frac{1 - \text{erf}(d_2/\sqrt{2})}{2}$ 誤差函數 $\text{erf}$ 投資等級公司 $d_2 > 1$，erf 輸入可能超出 [0, 1] 逼近區間

與 Black-Scholes 的不同之處：Merton 不需要 $\exp$（沒有股息折現因子）， 但對 $\ln$ 和 $\text{erf}$ 的精度要求更高——因為投資等級公司的 PD 差異在千分之幾， 2 個基點（0.02%）的誤差就可能翻轉合規判定。

信用違約交換（CDS）

基本結構

CDS 涉及三方：

  Protection Buyer               Protection Seller
  （信用保護買方）                  （信用保護賣方）
        |                               |
        |  ----  定期保費（spread）--->  |
        |  <--- 違約時賠償損失 ----      |
        |                               |
        +--- 標的：Reference Entity（參考實體，如某公司） ---+
  • 買方：持有公司債或暴露於信用風險的機構，希望轉移風險
  • 賣方：收取保費，承擔信用事件發生時的賠償責任
  • 保費（spread）：以基點（bps）計價，例如 100 bps = 每年支付名目本金的 1%
  • 信用事件：包含破產、未能付款、債務重組等

CDS Spread 與違約機率的關係

簡化近似：

$$\text{CDS spread} \approx PD \times LGD$$

其中 $LGD$（Loss Given Default）為違約損失率，通常假設 60%。

因此 CDS spread 直接反映市場對違約機率的定價。Merton 模型計算出的 PD 可以與 CDS 市場報價對比， 也可以用於內部風險管理。

CDS 組合的信用風險

持有多個 CDS（賣方部位）的機構，其加權違約機率為：

$$\text{Weighted PD} = \sum_{i=1}^{N} w_i \cdot PD_i, \quad \sum_{i=1}^{N} w_i = 1$$

監管要求此值不超過門檻值。這就是情境 C 的 ZK statement： 在不揭露每間公司的 $V_i, D_i, \sigma_i$ 的前提下，證明組合的加權 PD 合規。

為什麼需要零知識證明？

金融模型的輸入參數（持倉明細、隱含波動率、公司資產價值）是機構的核心機密。 現行的合規流程要求機構向監管單位完全揭露這些數據，產生了根本性的矛盾：

矛盾傳統做法ZK 做法
監管需要驗證合規 機構提交完整持倉數據 機構提交 ZK proof（~7 KB）
機構需要保護商業機密 被迫揭露策略細節 監管只看到 proof 是否驗過
驗證需要信任計算正確性 依賴審計、抽查 數學保證（soundness）
兩個情境的 ZK Statement 對比
情境 A：Black-Scholes情境 C：Merton
陳述 「我的選擇權組合 Delta 中性」 「我的 CDS 組合違約機率夠低」
公開 Merkle root, 門檻 $\epsilon$ $r$, $T$, 門檻 $\tau$
私密 $S, K_i, \sigma_i, w_i, T_i, q_i$ $V_i, D_i, \sigma_i, w_i$
關係 $|\sum w_i \cdot \Delta_i| \leq \epsilon$ $\sum w_i \cdot PD_i \leq \tau$, $\sum w_i = 1$

精度需求差異

兩個情境對計算精度的要求不同，這直接影響了 ZK 電路的設計：

情境 A 為何 pp=24 足夠

Black-Scholes Delta 的值域是 $[0, 1]$（call）或 $[-1, 0]$（put），合規門檻通常設在 $|\Delta_{\text{total}}| \leq 0.05$。 即使計算有 $10^{-5}$ 量級的誤差（約 22 bits 精度），距離 0.05 的判定邊界仍有足夠的安全餘裕。

情境 C 為何需要 pp=32+

投資等級公司的 PD 通常在 0.1%–2% 之間。以一個由 BBB 等級公司組成的 CDS 組合為例：

  • 加權 PD 可能在 1.5% 附近
  • 門檻設在 2%
  • 餘裕僅 0.5%（50 個基點）
  • 若 ln 或 erf 的計算誤差導致 PD 偏移 0.5% 以上，就會翻轉合規判定

因此需要至少 28 bits 的有效精度（pp=32 提供 28.1–29.0 bits）， 才能確保計算誤差不會影響合規結論。

erf 域限制

目前的 erf 多項式逼近區間為 $[0, 1]$，對應 $\Phi$ 的有效覆蓋範圍至 $\Phi(x)$ 中 $x \leq \sqrt{2}$， 即 PD $\geq$ 7.9%。投資等級公司（PD < 2%）的 erf 輸入超出此區間， 需要額外的域擴展技術（如分段逼近或漸近展開）。這是目前已知的主要技術限制。

函數 來源 情境 A 情境 C 電路中的角色

差異比較

與現有 ZK 超越函數計算方法的系統化比較

1產業與學術全景

1.1兩大範式

在零知識證明中計算超越函數（transcendental functions），現有方法可歸納為兩個根本不同的範式：查表法（Lookup-based）與多項式近似（Polynomial Approximation）。前者將函數值預先計算並存入大型查找表，後者則用低次多項式逼近目標函數。兩者在 table 成本、每次呼叫成本、以及適用範圍上各有取捨。

範式 原理 Table 成本 每次呼叫成本 適用函數
查表法 (Lookup-based) 預計算 f(x) 的所有可能值 大 (65K+/函數) 1 lookup 任意
多項式近似 (Polynomial) degree-d 多項式近似 f(x) 小 (256 共用) d 次乘法 + 驗證 需平滑、單調

1.2查表系統概覽

查表法是目前 ZK 超越函數計算的主流範式。以 ezkl 為代表，大多數系統將函數值預計算存入查找表（每函數 16K–65K entries），每次呼叫僅需一次 lookup，精度可達零誤差，但記憶體與 proving 開銷隨函數種類線性增長。

4逐項貢獻論證（摘要）

各貢獻的詳細論證、ablation 分析與逐系統定位比較見完整參考版 §4。以下僅列核心結果。

4.1Contribution 1：Minimax 係數選擇

同一電路（Horner deg-5）、同一表（256-entry）、同一域（BN254），僅替換 Taylor → Minimax 係數。窮舉驗證（156,390 合法輸入）結果：在固定電路條件下，Minimax 的精度改善從數十倍到數千倍不等，倍率取決於函數的 Taylor 收斂結構（詳見 §3.6 表格）。vs ZKFixedPointChip：已用 lolremez 但無 Taylor 對照、無 ULP 量化、無窮舉驗證。vs FHE：Minimax 是標準，但 ZK 需處理有限域量化、顯式截斷與對抗式 prover soundness。

4.2Contribution 2：Deferred Truncation

BN254 有 254-bit 域容量，而 Horner 中間值僅需 (d+1)×pp bit（如 d=5, pp=16 → 96 bit，剩餘 158 bits headroom）。將 d 次截斷合併為最終 1 次，避免 d−1 次累積誤差 → 2.0x 精度改善（三函數窮舉驗證：exp ULP 2→1、ln ULP 2→1、erf ULP 6→3），且 lookup 數量節省約 40%。前提：(d+1)×pp < 254。概念最近鄰為 IEEE 754 的 Compensated Horner / FMA，但從未在 ZK 有限域中實作。Kaizen/zkRNN 的延後是 DNN layer 粒度（非 Horner 內部），ZKFixedPointChip 承認 BN254 headroom 但未利用。

4.3Contribution 3：Universal Kernel

電路結構由 (pp, d) 參數化：6 gates, ⌈pp/8⌉ + ⌈d×pp/8⌉ lookups（R/w/gap 透過 selector union 共用 byte 欄位）, d+4 rows, 256-entry ByteTable。以 pp=16, d=5 為例：2+10 = 12 lookups。切換函數只需替換 d 個係數。實測 4 函數（exp, ln, erf, GELU）prove time 差異 < 1.4%，proof size 完全相同（4,992 B）。已驗證推廣至 13 個函數、546 個可行配置點（442 窮舉 + 104 取樣）。exp-ln 對偶性：共用 [0, ln2) 區間，兩者 d=5 均 ULP=1，僅 5 個係數不同。

4.4Function Coverage Criteria

提出充分條件 C1-C6（零錨點、嚴格單調、輸出有界、多項式可近似、係數量化、輸入分解），判定函數是否可由 universal kernel 處理。13 個常見函數（exp、ln、sin、1-cos、arcsin、arctan、tan、sinh、erf、GELU、sigmoid、1-exp(-x)、1-exp(-x²)）均滿足所有條件。Coverage 分析將系統從「支援特定函數的實作」提升為「可覆蓋任意滿足條件之函數的方法論」。

4.5寬區間驗證與有害次數效應

寬區間 Minimax 優勢：5 個飽和/組合函數在寬區間（未做範圍歸約）下 Minimax 贏 Taylor 139x–3,741x（如 erf [0,2.5] 的 3,741x）。Taylor 更出現大量溢出（w≥S 的輸入點數達 83K–381K）和負多項式輸出，直接導致 proof 無法生成。

有害次數效應 (Harmful Degree Effect)：寬區間上更高的 d 可能反而增加 ULP——如 sigmoid [0,8] 從 d=4 的 ULP=508 惡化至 d=5 的 ULP=14,230（28x 惡化）。原因是量化放大：高 d 導致整數係數位寬指數增長，量化誤差通過 Horner 鏈累積，壓倒逼近誤差的改善。據我們所知，此效應首次在 ZK 文獻中被記錄。設計意義：優先縮窄區間（範圍歸約）而非盲目提升 d。詳見完整參考版 §4.6。

4.6(pp, d) 設計空間全探索

首次對 (pp, d) 設計空間進行系統性探索：覆蓋 pp ∈ {12–36} × d ∈ {3–9} × 13 函數，共 637 個格點（546 可行 + 91 BN254 溢出排除），其中 pp≤28 為窮舉驗證。核心發現：

  • pp=16 + d≤7 是唯一覆蓋 13/13 函數的實用配置
  • pp=24 + d≤9 亦幾乎全數滿足 ULP=1：12/13 函數 ULP=1，僅 tan 為 ULP=2（<3）。但 pp=24 已是查表法實用邊界（dmax=10，BN254 空間緊張）——即使在此上限，方法仍近乎完美
  • d*=4 佔 6 函數、d*≤5 佔 10/13（77%）——d=5 是甜蜜點（12/13 達 ULP≤3）
  • 函數難度三群組：快速收斂（exp/sin 等，pp*>40）、中等（GELU/ln 等，30–40）、慢收斂（erf/tan 等，<30）
  • pp≥32 基本不可行（BN254 dmax≤7，僅 3/13 函數 ULP=1）
  • pp=16→24 成本：prove time 2.1x、proof size 1.38x（table 不變，仍 256）
  • 範圍歸約僅增加 36–50% 行數——比無歸約時的 ULP>1000 失敗，是高性價比投資

4.8Contribution 8：交叉驗證體系

所有數據經 3 個獨立路徑交叉驗證：Python Remez（150-bit）、Python 整數 Horner、Rust/Halo2 ZK 電路。499 個測試點全數一致（78 主要 + 48 補充 + 360 pp 敏感性 + 13 跨語言）。詳見完整參考版 §4.7。

5誠實的限制

1. ULP 範圍限制：max ULP claims 僅限於 Horner evaluation kernel 本身。End-to-end 精度取決於 range reduction 的實作品質——例如 exp 需要先將輸入分解為整數部分與小數部分。Range reduction 的精度損失不在本論文範圍內。
2. Batch 場景：N=1,000 時，我們的 prove time 為 1,011ms，而 Method B（Chunk lookup）僅需 135ms（7.5x 劣勢）。每次求值需要 9 rows，而查表法只需 1 row。在大量批次計算中，查表法的效率優勢明顯。
3. Proof size：4,992B vs Method A（Direct lookup）的 736B，大了 6.8x。雖然在絕對值上仍然可接受（不到 5KB），但並非最小。
4. Deferred truncation bit budget：需滿足 (d+1)×pp < 254（BN254 域容量限制）。pp=16 時 d_max=14（綽綽有餘），但 pp=32 時 d_max=6。超高精度場景需退回 per-step truncation 或使用部分延遲策略。

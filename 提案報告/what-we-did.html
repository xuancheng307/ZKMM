0. 一句話版本

我們沒有發明新的零知識證明系統。我們設計了一套電路層級的方法論，讓現有的 ZK 系統（Halo2）能精確計算 exp、ln、sin 等超越函數。類比：我們不是設計 CPU 的人，我們是寫高效組合語言程式的人。

1. 名詞定義

超越函數（Transcendental Function）

不能用有限次加減乘除和開根號表示的函數。嚴格定義：不存在任何非零的二元多項式 P(x, y)，使得 P(x, f(x)) = 0 對所有 x 成立。（注意：這是超越函數的定義，和超越數的定義不同。）

常見例子：exp(x)、ln(x)、sin(x)、cos(x)、tan(x)、arcsin(x)、arctan(x)、sinh(x)、erf(x)

不是超越函數的例子：x²+3x+1（多項式）、√x（代數函數）、1/(1+x)（有理函數）

我們論文涵蓋的 13 個函數：

類別函數
NN / ML 相關sigmoid, GELU, erf(x), exp(x)−1
標準數學函數sin, tan, arcsin, arctan, sinh, ln(1+x)
複合函數1−cos(x), 1−e−x, 1−e−x²

為什麼不需要「證明」它們是超越函數？這些都是經典數學中已知的超越函數。exp 的超越性由 Lindemann（1882）確立；sin、cos、ln、arctan 等的超越性也是 19 世紀就已知的結果。sigmoid = 1/(1+exp(−x)) 和 GELU（涉及 erf）是超越函數的組合，本身也是超越的。但更重要的是：我們論文真正需要的論點比「超越性」更簡單——這些函數不是多項式，所以不能用有限次域上加法和乘法精確計算，這才是我們需要近似方法的根本原因。

可擴展性：其他超越函數也能放進來嗎？

可以 我們的方法論是通用的。任何在有界區間上足夠平滑的函數都可以使用這套框架：

  1. 離線用 Remez 算法計算新函數的 Minimax 係數（幾秒鐘）
  2. 把係數載入同一個電路——同一 degree 下，電路結構、gate 數量、lookup 數量完全不變（若需要更高 degree，Horner gate 數和 lookup 數會相應增加）
  3. 用同一張 256 筆的 ByteTable

我們選擇這 13 個函數是因為它們覆蓋了 ZKML、金融計算、科學計算的常見需求，不是因為方法論只適用於這 13 個。唯一的限制是函數必須能被低次多項式（d ≤ 7 左右）在某個有界區間上精確近似。

定點數（Fixed-Point Number）

用整數模擬實數的方法。給定精度參數 pp，縮放因子 S = 2pp。實數 r 被表示為整數 R = round(r × S)。

例如 pp=16 時，S=65,536。實數 0.5 變成整數 32,768。加法是自然的（域上加法），但乘法後需要除以 S（「截斷」）來恢復尺度。

為什麼用定點數不用浮點數？ZK 電路操作的是有限域上的整數。浮點數需要指數和尾數的條件邏輯，在算術電路中極其昂貴。定點數只需加法和乘法，和電路原生操作匹配。

ULP（Unit in the Last Place）

誤差度量：電路輸出和理想值之間差了幾個最小可表示的定點數單位。

ULP(R) = |round(f(R/S) × S) − 電路輸出(R)|

ULP=0 表示完美，ULP=1 表示差一個最小量子（多項式近似方案在量化後的最佳可能精度；查表法可以做到 ULP=0），ULP=12 表示差了 12 個量子。我們關心的是所有合法輸入上的最大 ULP（worst-case）。

Minimax 多項式

在所有 d 次多項式中，使得「在整個區間 [a,b] 上的最大絕對誤差」最小的那一個。用 Remez 算法計算。

p* = argminp ∈ Pd maxx ∈ [a,b] |f(x) − p(x)|

直覺：Minimax 把誤差預算均勻分配到整個區間（L∞ 最優）。Taylor 把所有精度集中在展開點，犧牲端點。

Horner 求值法

把多項式 adxd + ... + a1x + a0 改寫成巢狀形式 (...((ad·x + ad-1)·x + ad-2)·x + ... )·x + a0。只需要 d 次乘法和 d 次加法。在 ZK 電路中，乘法 = gate，加法 = 免費。

2. 問題是什麼

核心問題：ZK 算術電路只能做有限域 Fp 上的加法和乘法。如何在這個限制下計算 exp(x)、ln(x)、sin(x) 等超越函數？

超越函數在實際應用中無處不在：

  • ZKML：神經網路需要 sigmoid、GELU、softmax（exp）
  • 可驗證金融計算：複利（exp）、Black-Scholes 定價（erf + exp + ln）
  • 可驗證科學計算：三角函數、雙曲函數

現有兩條路：

查表法多項式近似法
做法預算所有輸入→輸出的對應表，用 lookup argument 查表用低次多項式近似函數，在電路中用乘法和加法求值
表大小每函數 16K ~ 65K 筆共用 256 筆（我們的方法）
精度完美（ULP=0，但受限於表的解析度）ULP=1（在最優配置下）
代表系統ezkl, zkLLM, zkGPTzkTaylor, ZKFixedPointChip, 我們

我們走的是多項式近似這條路，但文獻中有四個未解決的缺口：用 Taylor（不是 L∞ 最優）、沒有 ULP 誤差分析、沒有 (pp,d) 設計空間探索、沒有延遲截斷技術。

沒有 我們完全沒有修改 ZK 證明系統的任何部分。

  • KZG 承諾方案（commitment、opening proof、batched verification）

我們做的是在 Halo2 提供的框架內設計電路。就像：

  • 我們寫的是「程式」（特定的 gate 配置和 lookup 表）
  • CPU 的正確性保證（= ZK 的 soundness/ZK 性質）在協議層由 Halo2/PLONK/KZG 提供；我們在電路層另外論證了 completeness、knowledge soundness 和 zero-knowledge（見 Section 6）

5. 我們實際做了什麼（三個貢獻）

貢獻 1：用 Minimax（而非 Taylor）多項式做近似

數值分析層面 我們把數值分析中已知最優的 Minimax 多項式（透過 Remez 算法計算）系統性地引入零知識電路設計，並首次提供了 Minimax vs Taylor 的受控比較和窮舉 ULP 分析。（注：ZKFixedPointChip 也用了 Minimax，但沒有 ULP 分析；zkTaylor 用的是 Taylor。）

我們做了什麼：

  • 用 Remez 算法在 50 位精度下計算 Minimax 多項式係數（c0=0 約束）
  • 設計 pre-scaling 量化方案：Di = round(ci × Sd+1−i)
  • 首次在 ZK 電路中做 Minimax vs Taylor 的受控比較

結果：在 13 個函數上，Minimax 比 Taylor 改善 2× ~ 4,330×。52 個配置中 29 個 Minimax 成功而 Taylor 完全失敗。

技術性質：這是數值逼近的應用，不是密碼學的貢獻。Minimax 逼近理論可追溯到 Chebyshev（1854），Remez 在 1934 年提出了高效計算 Minimax 多項式的算法。我們的貢獻不是發明新的逼近理論，而是把它引入 ZK 的定點數環境、提供窮舉 ULP 驗證、並量化了與 Taylor 的差距。

貢獻 2：延遲截斷（Deferred Truncation）

電路設計層面 利用 BN254 有限域的 254 位容量，把 Horner 求值鏈中的 d−1 次中間截斷合併為一次最終截斷。

標準做法：每一步乘法後都截斷（除以 S），引入 d−1 次捨入誤差，累積 O(d) ULP。

我們的做法：利用 BN254 的域足夠大這一事實，不做中間截斷，讓中間值在域中自然增長，最後一次截斷。

前提條件（Headroom Constraint）：

(d + 1) × pp < 254

這保證 Horner 累加器不會超出 BN254 域。pp=16, d=5 時只用 96 位，有 158 位餘量。

結果：13/13 函數在 d* 上必須用延遲截斷才能達到 ULP=1。逐步截斷全部失敗（2×~7× 誤差比率）。

技術性質：這是電路設計技巧。利用的是 BN254 域大於定點運算需求這個事實。不涉及新的密碼學假設。

貢獻 3：(pp, d) 設計空間分析

實驗分析層面 系統性探索 13 個函數在 7 個精度 × 7 個次數的格點上的精度和成本行為。

我們做了什麼：

  • 637 個格點（13 × 49），546 個有效配置（91 個 BN254 溢位）
  • 442 個窮舉驗證（pp ≤ 28，測試每一個整數輸入）
  • 104 個取樣驗證（pp ∈ {32, 36}，2M 個點）
  • 發現「量化飽和」現象：增加 d 可能反而惡化精度

技術性質：這是實驗性的系統化分析。提供了實用的設計指引（pp=16 時 d ≤ 7 覆蓋所有 13 函數），但不是理論證明。

6. 安全性：我們證明了什麼、沒證明什麼

這是最重要的一節。讓我們非常精確地區分。

我們的安全性宣稱（Security Claim）

兩層論證 我們的安全性分析分為協議層和電路層兩個層次：

  1. 協議層（繼承）：假設 Halo2/PLONK/KZG 是安全的，則沒有 PPT prover 可以為違反電路約束的 assignment 產生有效 proof。這是 PLONK 的標準保證，我們原封不動繼承。
  2. 電路層（我們的貢獻）：我們形式化論證我們設計的 6 個 gate 正確實現了三項安全性質——perfect completeness（誠實 prover 一定能構造滿足所有約束的 witness）、knowledge soundness（被提取的 witness 一定滿足輸入範圍、輸出範圍、正確 Horner 求值三個性質）、computational zero-knowledge（電路設計不引入額外資訊洩漏）。

直覺：協議層保證「約束被滿足 ⇒ proof 有效」，電路層保證「我們的約束 ⇔ 我們宣稱的性質」。兩層合在一起才是完整的安全論證。

展開來說，我們的電路約束保證三件事：

#保證由什麼約束強制安全等級
1 輸入 R ∈ [0, Rmax] ExplicitRC gate（gap 分解 + byte lookup） 確定性（不依賴機率論證）
2 輸出 w ∈ [0, S) 輸出 byte 分解 + ByteTable lookup 確定性
3 w 是正確的 Horner 求值結果（含延遲截斷） Horner gate × d + 截斷 gate 確定性（域上的多項式等式）

結合精度分析，這保證：|w − round(f(R/S) × S)| ≤ ULPmax，其中 ULPmax 是窮舉驗證的最大 ULP。

電路層安全論證（論文 ch3 §3.6）

我們對電路的 6 個 gate（horner_step、truncation、r_byte_decomp、w_byte_decomp、rem_byte_decomp、r_range_check）逐一論證了三項性質：

1. Perfect Completeness

對每一個合法輸入 R ∈ [0, Rmax]，誠實 prover 都能構造一個滿足所有電路約束的 witness assignment。逐約束論證：

  • ExplicitRC：R ≤ Rmax ⇒ gap = Rmax − R ∈ [0, 2pp)，可唯一分解為兩個 byte
  • Horner gates：誠實計算 Hk = Dk + R·Hk+1，中間值 ≤ 2(d+1)·pp。由 Headroom Constraint (d+1)·pp < 254，保證不會在 BN254 域中 wrap-around
  • Truncation gate：wpre = w·Sd + rem 是整數除法的恆等式，自然成立
  • Byte decompositions：R、w、rem 都是非負整數且在各自位寬範圍內，byte 分解必然成功

2. Knowledge Soundness

由 Halo2/KZG 的 knowledge soundness，對任何能產生被接受 proof 的 PPT prover，存在一個 PPT extractor 能輸出滿足所有電路約束的 witness（除 negligible probability 外）。我們論證被提取的 witness 必然滿足：

  1. R ∈ [0, Rmax] — 由 ExplicitRC 的 byte lookup 和 gap 等式強制（見下方 ExplicitRC 安全分析）
  2. w ∈ [0, S) — 由 w 的 byte 分解強制
  3. w 是正確的 Horner 求值 + 延遲截斷結果 — 由 d 個 Horner gate + truncation gate 聯合強制

結合誤差分析，保證 |w − round(f(R/S)·S)| ≤ ULPmax。

3. Computational Zero-Knowledge

我們驗證電路設計不引入超出 Halo2 保證的額外資訊洩漏：

  • Witness 放置：所有 prover 秘密值（R、Hk、wpre、w、rem、gap bytes、所有 byte 分解值）全部放在 advice columns。Halo2 在產生 KZG commitment 前會對 advice column 多項式加入隨機 blinding factor，在 DLOG 假設下是 computationally hiding
  • 公開資料：唯一的非 advice 資料是固定在 fixed column 的多項式係數 (D1, ..., Dd)、gate selectors、256 筆 ByteTable。這些是函數的公開參數（類比為「程式碼」），不是 prover 的秘密
  • 無 instance columns：電路不使用 instance columns，因此沒有 witness 值被暴露為 public input
  • 僅用標準原語：只用 custom gates 和 lookup arguments（Halo2 標準原語），沒有使用 raw polynomial openings 或 unblinded commitments 等非標準操作

我們明確「沒有」證明的東西（協議層）

協議層：繼承不證明 以下協議層的安全性質不是我們證明的，我們原封不動繼承：

  • PLONK 的 soundness — 來自 Gabizon et al. 2019
  • KZG 承諾的 binding 性質 — 基於 q-SDH 假設（Kate et al. 2010）
  • PLONK 的 simulator 構造（協議層的零知識）— 來自 PLONK 論文
  • BN254 曲線的安全性 — 基於 DLOG 和 pairing 假設
  • Halo2 實作的正確性 — 由 PSE 團隊維護
  • Fiat-Shamir 的安全性 — 來自 random oracle model
  • lookup argument 的 soundness — 來自 plookup（Gabizon & Williamson 2020）

我們繼承（inherit）了 Halo2/PLONK/KZG 的協議層安全性質，並在電路層補充了我們自己的安全論證。如果 Halo2 被攻破了，我們的系統也會被攻破——但那不是我們的論文的問題。

ExplicitRC 的安全分析

ExplicitRC（gap 分解）是我們自己設計的約束，所以它的安全性由我們分析：

我們自己分析的

約束：gap_lo + gap_hi × 256 + R = Rmax，其中 gap_lo, gap_hi ∈ [0, 255]（由 ByteTable lookup 強制）。

安全論證：如果 R > Rmax，則 Rmax − R 在 Fp 中是 p − (R − Rmax)，一個接近 p 的巨大數字（> 2253），無法表示為兩個 byte 的線性組合（最大 255 + 255×256 = 65,535）。byte lookup 必然失敗。

性質：確定性（deterministic）。不依賴機率論證或計算假設。只依賴有限域的代數性質和 lookup argument 的 soundness。

我們也分析了一個被拒絕的替代方案（ORV）：利用多項式單調性隱式驗證輸入範圍。分析發現 13 個函數中有 7 個存在安全漏洞（wmax < S−1 導致的間隙），所以我們採用了更保守但更安全的 ExplicitRC。

7. 我們有完整證明 ZK 嗎？

有，兩層合起來是完整的

協議層（PLONK/KZG 繼承）+ 電路層（我們的論證，見 Section 6）合在一起，三項性質都有覆蓋：

9. 數值分析的貢獻 vs 密碼學的貢獻

一個合理的質疑是：「這看起來更像數值分析的論文，對密碼學的貢獻在哪裡？」

誠實的回答：核心技術創新確實來自數值分析（Minimax、Horner、Remez 都是幾十年前的經典方法）。但貢獻的價值在於跨域的橋梁：

純數值分析的部分

  • Minimax 多項式本身（Remez 1934）
  • Horner 求值法（Horner 1819）
  • 多項式逼近的誤差理論

這些都是教科書內容，不是我們的貢獻。

只有在 ZK 環境才有意義的部分

技術為什麼只在 ZK 中有意義普通 CPU 上的情況
延遲截斷 利用 BN254 的 254-bit 域容量，讓中間值在域中自然增長。這在普通 CPU 上不需要（有 64-bit float 和 FMA 指令處理精度） 不需要——浮點硬體自動處理
定點數量化方案 ZK 電路沒有浮點數，必須用定點數。係數量化策略 Di = round(ci × Sd+1−i) 是為 Horner + 延遲截斷設計的 不需要——直接用 double
ExplicitRC 在 ZK 中 prover 可以作弊餵假輸入，必須有約束強制輸入範圍。普通計算不需要——你直接 if-else 檢查 不需要——一行 if 解決
量化飽和現象 高次係數在定點量化後相對誤差大，乘以 xd 被放大。浮點數沒有這個問題 不存在——浮點數有均勻的相對精度（固定有效位數），不受量化放大效應影響
(pp, d) 設計空間 pp（精度）和 d（次數）的交互效應只在定點數 ZK 環境才出現。受 BN254 容量 (d+1)×pp < 254 硬性約束 不存在——沒有域大小限制

對密碼學 / ZK 社群的具體貢獻

我們的論文對 ZK 社群的價值不是「發明新密碼學」，而是：

  1. 提供一個實用的 ZK gadget：電路設計者需要算超越函數時，除了查表現在有第二條路。特別是高精度場景（pp ≥ 28 時查表需要 2.68 億筆，不可行）和多函數場景（13 個函數共用 256 筆 vs 每個函數 65K 筆）
  2. 電路層的完整安全論證：形式化論證 completeness（誠實 prover 必定通過）、knowledge soundness（通過的 witness 必定合法）、zero-knowledge（不洩露秘密）三項性質。加上 ExplicitRC 的確定性正確性證明和 ORV 的漏洞發現。這是寫 ZK 電路的人需要、但文獻中缺乏的安全分析
  3. 設計指引：(pp, d) 設計空間分析告訴電路設計者哪些配置可行、哪些會失敗。在我們之前，沒有人系統性做過這件事
  4. 通用框架：換一個函數只需換係數，不需重新設計電路。這降低了 ZK 應用開發的門檻

類比：就像「高效 SHA-256 電路實作」也不是新密碼學，但對 ZK 社群有實用價值。我們的工作性質相同——不是密碼學理論，是密碼學工程（cryptographic engineering），解決的是「怎麼在 ZK 電路裡高效算超越函數」這個工程問題。
